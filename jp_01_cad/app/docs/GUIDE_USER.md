## 1. CAD Vision이란?

CAD Vision은 **AI 기반 도면 검색 시스템**입니다.

| 기능 | 설명 |
|---|---|
| 📊 **대시보드** | 등록된 전체 도면 현황을 한눈에 확인 |
| 📁 **도면 등록** | 새로운 도면 파일을 시스템에 등록 (AI 자동 분류) |
| 🔍 **도면 검색** | 텍스트로 원하는 도면을 빠르게 검색 |
| 🤖 **도면 분석** | AI가 도면을 분석하여 부품 정보를 추출 (환각 검증 포함) |

**지원 파일 형식**: PNG, JPG, JPEG, PDF, TIFF

## 2. 설치 전 준비사항

CAD Vision을 실행하려면 **Docker Desktop**이 필요합니다.

### Docker Desktop 설치

| 운영체제 | 다운로드 링크 |
|---|---|
| **macOS** | https://www.docker.com/products/docker-desktop/ |
| **Windows** | https://www.docker.com/products/docker-desktop/ |

#### macOS 설치 순서

1. 위 링크에서 **Download for Mac** 클릭
2. 다운로드된 `Docker.dmg` 파일 실행
3. Docker 아이콘을 Applications 폴더로 드래그
4. Applications에서 **Docker** 실행
5. 상단 메뉴바에 🐳 고래 아이콘이 나타나면 준비 완료

#### Windows 설치 순서

1. 위 링크에서 **Download for Windows** 클릭
2. 다운로드된 설치 파일 실행
3. 설치 마법사 안내를 따라 진행
4. 설치 완료 후 **재부팅**
5. Docker Desktop 실행
6. 작업표시줄에 🐳 고래 아이콘이 나타나면 준비 완료

> **참고**: Docker Desktop은 무료로 사용 가능합니다 (개인/소규모 기업).

### 시스템 요구사항

| 항목 | 최소 사양 |
|---|---|
| RAM | 8GB (16GB 권장) |
| 디스크 여유 공간 | 20GB |
| 외장 드라이브 | 프로젝트가 저장된 외장 드라이브 연결 필수 |
| 인터넷 | 최초 설치 시 필요 (이후 오프라인 사용 가능) |

## 3. 설치 방법 (최초 1회)

### 3단계로 끝나는 설치

#### 1단계: 프로젝트 파일 받기

프로젝트 폴더를 전달받은 후 외장 드라이브에 저장합니다.

현재 저장 위치: `/Volumes/Corsair EX300U Media/00_work_out/01_complete/me/01_CAD/drawing-llm`

> **중요**: 프로젝트가 외장 드라이브에 저장되어 있으므로, 실행 전에 반드시 **외장 드라이브가 연결**되어 있어야 합니다.

#### 2단계: 터미널 열기

| 운영체제 | 터미널 여는 방법 |
|---|---|
| **macOS** | `Spotlight (⌘+스페이스)` → "터미널" 입력 → Enter |
| **Windows** | `시작 메뉴` → "PowerShell" 검색 → 실행 |

#### 3단계: 실행 스크립트 실행

터미널에 아래 명령어를 **한 줄씩** 입력합니다:

```bash
# 프로젝트 폴더로 이동 (외장 드라이브)
cd "/Volumes/Corsair EX300U Media/00_work_out/01_complete/me/01_CAD/drawing-llm"

# 실행 (최초 설치 시 5~15분 소요)
./scripts/docker-start.sh
```

> **경로에 공백이 있으므로** 반드시 **큰따옴표(`"`)로 감싸야** 합니다.

> **Windows 사용자**: 외장 드라이브 문자를 확인한 후 PowerShell에서 아래 명령어를 사용하세요:
> ```powershell
> # 외장 드라이브 경로로 이동 (드라이브 문자는 환경에 맞게 수정)
> cd "E:\00_work_out\01_complete\me\01_CAD\drawing-llm"
> docker compose up -d --build
> # Ollama가 시작될 때까지 30초 정도 대기 후 실행
> docker compose exec ollama ollama pull qwen3.5:9b
> # 브라우저 열기
> Start-Process "http://localhost:8501"
> ```

스크립트가 완료되면 **자동으로 브라우저가 열립니다**.

## 4. 실행 방법

### 최초 실행 이후 — 매번 실행할 때

1. **외장 드라이브를 연결합니다** (Finder에서 외장 드라이브가 보이는지 확인)

2. **Docker Desktop을 실행합니다** (상단/하단 메뉴바에 🐳 아이콘 확인)

3. **터미널에서 실행 스크립트를 실행합니다**:

```bash

```

4. **브라우저에서 접속합니다**: http://localhost:8501

> 💡 **팁**: 2회차부터는 AI 모델 다운로드가 생략되어 **1~2분 내**에 실행됩니다.

## 5. 화면 사용법

CAD Vision은 왼쪽 사이드바에서 4개 페이지를 전환할 수 있습니다.

### 📊 대시보드

시스템의 전체 현황을 보여줍니다.

- **등록된 도면 수**: 현재 시스템에 저장된 전체 도면 수
- **카테고리별 통계**: 도면 종류별 분포
- **최근 등록 도면**: 최근에 추가된 도면 목록
- **Ollama 상태**: AI 서버 연결 상태 (🟢 온라인 / 🔴 오프라인)

### 📁 도면 등록

새로운 도면 파일을 시스템에 등록합니다.

**단일 등록**:
1. "파일 선택" 버튼 클릭
2. 도면 파일 선택 (PNG, JPG, PDF 등)
3. 메타데이터(제목, 카테고리 등) 입력 (선택사항)
4. "등록" 버튼 클릭

**일괄 등록**:
1. "일괄 등록" 탭 선택
2. 도면이 저장된 폴더 경로 입력
3. "일괄 등록 시작" 클릭
4. 진행률 표시줄을 확인하며 대기

### 🔍 도면 검색

등록된 도면 중에서 원하는 도면을 검색합니다.

**텍스트 검색**:
1. 검색창에 키워드 입력 (예: "M8 볼트", "플랜지", "베어링")
2. Enter 또는 "검색" 버튼 클릭
3. 유사도 순으로 결과 표시

**검색 예시**:
- `"M8 육각볼트"` → M8 규격 볼트 관련 도면
- `"SUS304 플레이트"` → 스테인리스 플레이트 도면
- `"미스미 리니어 가이드"` → 미스미 리니어 가이드 도면

### 🤖 도면 분석

AI가 업로드한 도면을 분석하여 정보를 추출합니다.

1. "파일 선택"으로 분석할 도면 업로드
2. "분석 시작" 클릭
3. AI가 도면에서 다음 정보를 추출합니다:
   - 부품명 / 규격
   - 재질 / 치수
   - 특이사항

> **참고**: 도면 분석은 AI 서버(Ollama)가 연결된 상태에서만 사용 가능합니다.
> 대시보드에서 Ollama 상태가 🟢 온라인인지 확인하세요.

## 7. 자주 묻는 질문 (FAQ)

### Q. "Docker가 실행되고 있지 않습니다" 오류가 나옵니다

**A.** Docker Desktop을 먼저 실행해주세요.
- macOS: Applications → Docker 실행 → 상단 메뉴바에 🐳 아이콘이 나타날 때까지 대기
- Windows: 시작 메뉴 → Docker Desktop 실행 → 작업표시줄에 🐳 아이콘 확인

---

### Q. 설치에 시간이 오래 걸립니다

**A.** 최초 설치 시에는 다음 항목을 다운로드하므로 **10~15분** 소요될 수 있습니다:
- Python 앱 이미지 + 라이브러리 (~2GB)
- AI 모델 파일 (~6.6GB)

인터넷 속도에 따라 차이가 있으며, **2회차부터는 1~2분**이면 실행됩니다.

---

### Q. "port is already allocated" 오류가 나옵니다

**A.** 포트 8501 또는 11434가 이미 사용 중입니다.
1. 이전에 실행한 CAD Vision이 남아있을 수 있습니다
2. 아래 명령어로 정리 후 다시 실행하세요:

```bash
docker compose down
./scripts/docker-start.sh
```

---

### Q. 도면 분석이 동작하지 않습니다 / Ollama 🔴 오프라인

**A.** AI 서버(Ollama)의 상태를 확인하세요:
1. 대시보드 사이드바에서 Ollama 상태 확인
2. 🔴 오프라인이면 아래 명령어를 **순서대로** 실행:

```bash
cd "/Volumes/Corsair EX300U Media/00_work_out/01_complete/me/01_CAD/drawing-llm"

# Ollama 컨테이너 재시작
docker compose restart ollama

# 1~2분 대기 후, 앱 컨테이너도 재시작
docker compose restart app
```

3. 브라우저에서 http://localhost:8501 새로고침 후 대시보드에서 🟢 온라인 확인

> **참고**: 앱이 시작될 때 Ollama 연결 정보를 캐싱하므로,
> Ollama를 재시작한 후에는 **앱도 함께 재시작**해야 할 수 있습니다.

---

### Q. 도면 분석에서 "LLM 호출 실패 (500)" 오류가 나옵니다

**A.** AI 모델이 아직 로딩 중이거나 메모리가 부족한 경우 발생합니다.

1. **잠시 대기 후 재시도**: 모델 첫 로딩 시 30초~1분 소요. 시스템이 자동으로 최대 2회 재시도합니다
2. **반복 실패 시** 아래 명령어로 Ollama 재시작:

```bash
cd "/Volumes/Corsair EX300U Media/00_work_out/01_complete/me/01_CAD/drawing-llm"
docker compose restart ollama
```

3. 화면에 표시된 **오류 메시지**를 확인하세요:
   - "연결할 수 없습니다" → Ollama 서버가 꺼져 있음
   - "설치되어 있지 않습니다" → AI 모델 미설치 (`docker compose exec ollama ollama pull qwen3.5:9b`). 참고: Ollama 모델은 시스템 RAM에 따라 자동 선택됩니다 (48GB 이상: 27b, 16GB 이상: 9b, 16GB 미만: 4b)
   - "시간 초과" → 모델 로딩 중, 1~2분 대기 후 재시도
   - "out of memory" → Docker Desktop 메모리 설정 확인 (최소 8GB 권장)

---

### Q. 데이터를 다른 컴퓨터로 옮기고 싶습니다

**A.** `data` 폴더 전체를 복사하면 됩니다:

1. 원본 컴퓨터의 `drawing-llm/data/` 폴더를 USB 등으로 복사
2. 새 컴퓨터의 `drawing-llm/data/` 위치에 붙여넣기
3. 새 컴퓨터에서 `./scripts/docker-start.sh` 실행

---

### Q. 검색 결과에 이미지가 보이지 않습니다

**A.** 도면 이미지 파일의 위치를 확인하세요.

도면이 프로젝트 **외부 경로**(예: 외장 드라이브, 별도 폴더)에 저장된 경우, Docker 컨테이너에서 접근할 수 없어 이미지가 표시되지 않습니다.

**해결 방법**: 개발 담당자에게 문의하여 `docker-compose.yml`에 도면 경로 마운트 설정을 요청하세요.

> 참고: 이미지가 보이지 않더라도 **도면 메타데이터(파일명, OCR 텍스트, 부품번호 등)는 정상 표시**됩니다.
> 검색 기능 자체는 이미지 유무와 관계없이 동작합니다.

---

### Q. 검색 결과가 정확하지 않습니다

**A.** 도면이 올바르게 등록되었는지 확인하세요:
- 도면 이미지가 너무 작거나 흐린 경우 OCR 품질이 떨어질 수 있습니다
- 고해상도(300DPI 이상) 이미지를 사용하면 검색 정확도가 향상됩니다
- 도면 등록 시 AI가 자동으로 **81개 카테고리** 중 하나로 분류합니다 (정확도 93.87%)

---

### Q. "도면 분석" 결과가 실제와 다릅니다

**A.** AI가 생성한 분석 결과는 자동으로 **환각 검증**을 거칩니다.
- OCR로 추출한 부품번호, 재질, 치수를 AI 응답과 대조하여 일치 여부를 확인합니다
- 불일치가 발견되면 해당 항목이 경고로 표시됩니다
- AI 분석 결과는 참고용이며, 최종 판단은 엔지니어가 해야 합니다

---

### Q. 시스템을 완전히 삭제하고 싶습니다

**A.** 아래 순서로 진행하세요:

```bash
# 1. 컨테이너 + 볼륨 삭제

# 2. Docker 이미지 삭제

### Q. 외장 드라이브를 연결했는데 실행이 안 됩니다

**A.** 외장 드라이브가 정상적으로 마운트되었는지 확인하세요:

1. **Finder**에서 좌측 사이드바에 외장 드라이브 이름이 보이는지 확인
2. 터미널에서 아래 명령어로 확인:

```bash
ls "/Volumes/Corsair EX300U Media/00_work_out/01_complete/me/01_CAD/drawing-llm"
```

파일 목록이 보이면 정상입니다. "No such file or directory" 오류가 나오면 드라이브가 마운트되지 않은 것입니다.

> **팁**: 외장 드라이브를 연결한 직후에는 macOS가 마운트하는 데 몇 초가 걸릴 수 있습니다.
> Finder에서 드라이브가 보인 후에 터미널 명령어를 실행하세요.

### Q. 외장 드라이브를 안전하게 분리하려면?

**A.** 반드시 아래 순서를 지키세요:

1. 먼저 **Docker 컨테이너를 종료**합니다:
```bash
cd "/Volumes/Corsair EX300U Media/00_work_out/01_complete/me/01_CAD/drawing-llm"
docker compose down
```
2. Finder에서 외장 드라이브 옆의 ⏏ **추출 버튼**을 클릭
3. "디스크가 안전하게 제거되었습니다" 메시지 확인 후 물리적으로 분리

> **주의**: Docker 컨테이너가 실행 중일 때 드라이브를 분리하면 데이터 손상이 발생할 수 있습니다.
