## ④ 플랫폼별 반영 절차

Flutter 단일 코드베이스라 **구현은 공통**, 빌드·확인만 플랫폼별로 다르다.

1. **구현 권위**: 해당 화면 가이드(`0X-*.md`)의 ④ 체크리스트. '백엔드 공백' 명시 항목은 임의 구현 금지
2. **검증 게이트**: `flutter analyze` 0건 + `flutter test` 전통과(현 358) + 신규 문구 금칙어(진단/처방/치료/효능)·신뢰도 % 가드 테스트
3. **Android (Android Studio / Pixel 10 Pro)**:
   ```bash
   export ANDROID_SDK_ROOT=~/Library/Android/sdk   # 에뮬레이터 실행 시 필수
   flutter run --no-resident -d emulator-5556 --flavor dev   # API 기본 10.0.2.2
   ```
   ⚠ flavor 미지정 시 "No application found for TargetPlatform.android_arm64" — **반드시 `--flavor dev`**
4. **iOS (Xcode / iPhone 17 Pro·iOS 26.5)**: 시뮬레이터는 **반드시 UDID로 지정** — 동명 iPhone 17 Pro가 26.3/26.4 런타임에도 존재해 이름 선택 시 잘못된 기기에 설치된다
   ```bash
   flutter run --no-resident -d 7B2E1A72-B3C9-4102-8E3C-0009CCBFB4FB \
     --dart-define=LEMON_API_BASE_URL=http://127.0.0.1:8000/api/v1
   ```
5. **백엔드 dev 스택**: `docker compose up -d db backend` (`PERSIST_DAILY_HEALTH_SCORE` 기본 true)
6. **시안 대조**: 본 문서 frame id로 Figma 프레임을 열어 나란히 비교. 미세 색·간격은 design_tokens_v2가 단일 출처(hex 단독 사용 금지)
