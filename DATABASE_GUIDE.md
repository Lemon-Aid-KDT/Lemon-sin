## 1. 데이터베이스란 무엇인가?

앱을 사용할 때 우리가 입력하는 모든 정보(회원가입 정보, 식단 기록, 영양제 정보 등)는 어딘가에 저장되어야 합니다. 그 저장 공간이 **데이터베이스(DB)** 입니다.

비유하자면:

```
Excel 파일 = 데이터베이스
시트(탭)   = 테이블
행(row)    = 한 명의 사용자 정보 또는 한 건의 기록
열(column) = 이름, 이메일, 나이 같은 각각의 항목
```

## 2. 왜 PostgreSQL을 선택했는가?

### 비교표

| 항목 | Oracle | MySQL | **PostgreSQL** |
|------|--------|-------|----------------|
| 비용 | 유료 (매우 비쌈) | 무료 | **무료** |
| 기업 신뢰도 | 매우 높음 | 높음 | **매우 높음** |
| JSON 저장 | 가능 | 약함 | **매우 강력 (JSONB)** |
| 시계열 확장 | 별도 구매 | 없음 | **TimescaleDB 확장** |
| 행 단위 보안 | 가능 | 불가 | **RLS 지원** |
| Oracle 문법 유사도 | 기준 | 낮음 | **높음** |

### 선택 이유 3가지

**① 무료이면서 Oracle과 문법이 가장 비슷하다**
회사(레몬헬스케어)는 Oracle을 사용하지만, 학생 프로젝트에서 Oracle은 비용이 발생합니다. PostgreSQL은 무료이면서 Oracle SQL 문법과 가장 유사합니다.

**② 영양제/음식 데이터를 유연하게 저장할 수 있다 (JSONB)**
영양제 성분표는 제품마다 항목이 달라서 고정된 열(column)로 저장하기 어렵습니다. PostgreSQL의 JSONB 타입을 사용하면 자유로운 형태로 저장할 수 있습니다.

```
일반 방식 (불편):
  비타민C_함량 | 비타민D_함량 | 오메가3_함량 | ...
  (모든 영양소마다 열을 만들어야 함)

JSONB 방식 (유연):
  nutrients: {"비타민C": "500mg", "비타민D": "1000IU", "오메가3": "1g"}
```

**③ 시계열 데이터(걸음수, 심박수)를 효율적으로 처리한다**
하루에도 수천 건씩 쌓이는 걸음수·심박수 데이터를 일반 DB로 처리하면 느려집니다. TimescaleDB 확장을 추가하면 시간 순서 데이터를 최적화된 방식으로 저장합니다.

## 4. 파일 간 연결 구조

앱에서 로그인 버튼을 누르면 어떤 파일들이 동작하는지 순서대로 설명합니다.

```
[Flutter 앱]
    │
    │  POST /api/v1/auth/login
    │  {"email": "...", "password": "..."}
    ▼
[backend/src/main.py]          ← FastAPI 앱 시작점, 모든 요청의 입구
    │
    ▼
[backend/src/api/__init__.py]  ← 어느 URL을 어느 파일이 처리할지 연결
    │
    ▼
[backend/src/api/auth.py]      ← 로그인 처리 담당
    │
    ├─→ [backend/src/schemas/auth.py]    ← 요청 데이터 형식 검증
    │       "이메일 형식 맞아?" "비밀번호 있어?"
    │
    ├─→ [backend/src/db/session.py]      ← DB 연결 통로 (문지기)
    │       "DB에 연결할게요"
    │
    ├─→ [backend/src/models/user.py]     ← users 테이블 설명서
    │       "users 테이블이 어떻게 생겼는지 알려줘"
    │
    ├─→ [backend/src/utils/security.py]  ← 암호화·토큰 처리
    │       "비밀번호 맞는지 확인해줘" "JWT 토큰 만들어줘"
    │
    └─→ [PostgreSQL DB]                  ← 실제 데이터 저장소
            users 테이블에서 이메일로 검색
            비밀번호 일치 확인
            refresh_tokens 테이블에 토큰 저장
```

### 프로필 저장 흐름

```
[Flutter 앱]
    │
    │  PUT /api/v1/profile
    │  Authorization: Bearer {access_token}
    │  {"age": 52, "gender": "M", "height_cm": 178, ...}
    ▼
[backend/src/api/profile.py]   ← 프로필 처리 담당
    │
    ├─→ [backend/src/utils/deps.py]       ← "이 사람 로그인한 사람 맞아?"
    │       JWT 토큰으로 사용자 신원 확인
    │
    ├─→ [backend/src/models/profile.py]   ← profiles 테이블 설명서
    │
    └─→ [PostgreSQL DB]
            profiles 테이블에 저장
```

## 6. DB 연결 설정 파일

### 환경변수 (.env 파일)

```
POSTGRES_USER=lemon          ← DB 접속 아이디
POSTGRES_PASSWORD=lemon1234  ← DB 접속 비밀번호
POSTGRES_DB=lemon_aid        ← 사용할 DB 이름
POSTGRES_HOST=localhost      ← DB 서버 주소
POSTGRES_PORT=5432           ← DB 포트 번호

JWT_SECRET=your-secret-key   ← JWT 토큰 암호화 키
ACCESS_TOKEN_EXPIRE_MINUTES=30   ← 로그인 토큰 유효시간 (30분)
REFRESH_TOKEN_EXPIRE_DAYS=7      ← 갱신 토큰 유효시간 (7일)
```

### Docker로 DB 실행

```bash
# 이 명령어 하나로 PostgreSQL + Redis 동시 실행
docker-compose up -d
```

Docker는 **가상의 컴퓨터 안에 DB를 실행**하는 도구입니다. 내 컴퓨터에 PostgreSQL을 직접 설치하지 않아도 됩니다.

## 8. 보안 처리 방식

### 비밀번호
사용자가 입력한 비밀번호는 **절대 그대로 저장하지 않습니다.**
bcrypt 알고리즘으로 복호화 불가능한 해시값으로 변환하여 저장합니다.

```
입력: "mypassword123"
저장: "$2b$12$abcdefghijk..." (원래 비밀번호로 되돌릴 수 없음)
```

### JWT 토큰
로그인 성공 후 발급되는 두 가지 토큰:

| 토큰 종류 | 유효시간 | 용도 |
|-----------|----------|------|
| access_token | 30분 | API 호출 시 신원 증명 |
| refresh_token | 7일 | access_token 재발급 |

### 민감정보 (만성질환, 복약정보)
현재는 JSONB로 저장 중이며, **추후 AES-256 암호화 예정**입니다.
AES-256은 은행에서 사용하는 수준의 암호화 방식입니다.
