# Lemon PR 준비

커밋, push, PR 생성 전에 이 skill을 사용합니다.

## 체크리스트

- `git status --short`를 확인했습니다.
- diff가 사용자 요청과 직접 관련된 변경만 포함합니다.
- 관련 문서를 갱신했거나 해당 없음을 명시했습니다.
- 건강, 개인정보, 동의, 프롬프트, Tool, Agent 변경은 안전 리뷰를 완료했습니다.
- 검증을 실행했거나 건너뛴 이유를 문서화했습니다.
- PR 요약에는 변경 이유, 변경 표면, 검증, 영향 범위, 리뷰어가 볼 파일을 포함합니다.

## 커밋 기본값

- Conventional Commits를 사용합니다.
- 가능하면 300줄 이하의 작은 PR을 선호합니다.
- `git commit` 또는 `git push` 전에는 사용자에게 확인합니다.
