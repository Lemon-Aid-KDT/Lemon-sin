LEMON AID WEEK 02 TASK SOLUTION REPORT

음식·영양제 과제 해결 전략 보고서

기준 기간: 2026.05.11 - 2026.05.17
목적: 과제별 해결 구조와 기술 선택 이유 정리

음식 과제와 영양제 과제는 입력 데이터, 오인식 원인, 검증 기준이 다르므로 저장 전 preview 흐름을 분리한다. 두 흐름은 사용자가 확인한 뒤에만 confirmed health record 기준으로 DB에서 연결되며, 그 전까지 OCR·AI 결과는 후보와 근거로만 취급한다.

01

두 과제를 분리해서 푸는 이유

구분 음식 과제 영양제 과제
입력 불확실성 사진 각도, 여러 음식 동시 노출, 조리 상태, 실제 섭취량이 결과를 흔든다. 라벨 OCR 오인식, 성분명 표기 차이, 함량·단위 누락 또는 줄바꿈이 결과를 흔든다.
검증 방식 식품 후보와 100g 기준 영양 프로필을 맞춘 뒤 사용자가 양을 보정해야 한다. 성분명, 함량, 단위를 원문 근거와 함께 보여주고 사용자가 확인해야 한다.
안전 경계 섭취 기록 후보이며, 건강 판단이나 식단 처방처럼 표현하지 않는다. 주의 후보는 확정 판단이 아니며, 필요 시 전문가 상담을 권장하는 정보 제공으로 둔다.

음식 과제 해결 방식

해결해야 하는 문제는 사진 한 장에서 바로 섭취 기록을 확정하는 것이 아니라, 사용자가 확인할 수 있는 음식 후보와 영양 계산 근거를 만드는 것이다.

사진 인식음식 후보, confidence, 복수 후보를 분리한다.
영양 매칭100g 기준 영양 프로필과 출처를 연결한다.
섭취량 보정사용자 입력량으로 kcal와 영양소를 재계산한다.
확인 후 저장사용자 확인 뒤 섭취 기록 후보를 저장한다.

사용 기술과 이유

  • 이미지 모델은 음식 후보를 빠르게 좁히기 위한 1차 분류 도구로 사용한다.
  • 100g 기준 데이터셋은 모델 출력보다 재현 가능한 영양 계산 근거로 둔다.
  • preview 구조는 사진 기반 입력의 불확실성을 사용자가 보정할 수 있게 한다.

영양제 과제 해결 방식

해결해야 하는 문제는 라벨 이미지를 복용 기록으로 바로 바꾸는 것이 아니라, OCR 결과를 성분 후보와 원문 근거가 있는 검토 대상으로 구조화하는 것이다.

라벨 OCR제품명, 성분 영역, 원문 텍스트 후보를 추출한다.
성분 구조화성분명, 함량, 단위, confidence를 분리한다.
주의 후보복약·질환 관련 가능성을 검토 대상으로 표시한다.
확인 후 저장사용자 확인 뒤 복용 기록 후보를 저장한다.

사용 기술과 이유

  • OCR은 라벨 원문을 확보하기 위한 도구이며, 오인식 가능성을 전제로 둔다.
  • LLM 또는 parser 후처리는 성분 후보 구조화에 한정하고, 확정 판단으로 쓰지 않는다.
  • preview 구조는 OCR 오인식과 건강 문구 리스크를 저장 전에 차단하기 위해 둔다.
02

DB 병합 기준

Food Preview

음식 후보, 100g 영양값, 섭취량 보정값, 사용자가 수정한 값을 별도 preview로 유지한다.

Supplement Preview

OCR 원문 근거, 성분명·함량·단위 후보, 주의 후보를 별도 preview로 유지한다.

Confirmed Health Record

user_confirmed 이후에만 사용자, 기록 유형, 원문 근거, 확인 상태, audit 기준으로 DB에서 연결한다.

핵심 기준: preview 단계에서는 음식과 영양제를 합치지 않는다. 사용자 확인 전 결과는 저장 기록, 알림, Agent action의 근거로 확정하지 않는다.

03

현재 상태와 남은 결정

구현 상태

주간 기록 기준으로 인증, OCR provider 비교, 영양제 preview 모델, Agent 검증 계약이 병렬로 정리되었다. 한 화면 end-to-end 완료로 과장하지 않는다.

검증 상태

음식은 100장 수준 이미지 PoC, 영양제는 실제 라벨 OCR E2E, 공통은 preview -> confirmed -> saved 상태 계약 검증이 필요하다.

다음 결정

초기 음식 class 범위, OCR provider와 외부 전송 동의, raw OCR/provider payload 저장 정책, 안전 문구 검수 기준을 고정해야 한다.

04

다음 주 검증 기준

영역 검증할 흐름 완료 기준
음식 100장 이미지 PoC로 음식 후보, 영양 프로필 매칭, 섭취량 보정, 사용자 확인을 점검한다. 후보 실패와 낮은 confidence가 preview에서 분리되고, 사용자가 수정할 수 있다.
영양제 실제 라벨 이미지로 OCR, 성분 후보 추출, 함량·단위 정규화, 주의 후보 표시를 점검한다. 원문 근거와 후보가 함께 표시되고, 사용자 확인 전 저장 기록으로 확정되지 않는다.
공통 preview, user_confirmed, saved 상태 전환을 UI, API, DB, Agent 입력에서 같은 의미로 사용한다. 확인되지 않은 후보는 side effect를 만들지 않고, 확인 이력은 audit 기준으로 남는다.
안전 raw OCR text, provider payload, 이미지, 건강 정보의 저장·로그·삭제 정책과 문구를 검수한다. 진단·치료·처방처럼 보이는 단정 표현 없이 정보 제공과 전문가 상담 권장 경계를 유지한다.
