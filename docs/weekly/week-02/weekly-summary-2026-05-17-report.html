LEMON AID WEEK 02 REPORT

음식·영양제 입력 파이프라인 주간 보고

기준 기간: 2026.05.11 - 2026.05.17
근거 Daily: 05.13 - 05.17

이번 주 핵심은 이미지 인식 기능 자체보다, 음식 사진과 영양제 라벨 사진을 후보·근거·확인 상태를 가진 건강 데이터로 전환하는 기반을 정리한 것이다. 두 입력 도메인은 분리하되, 사용자 확인 전에는 기록과 Agent 입력으로 확정하지 않는 기준을 공통으로 둔다.

01
음식 입력

음식 사진은 식품 후보를 찾고, 100g 기준 영양 프로필과 사용자 섭취량 보정을 거쳐 섭취 기록 후보로 전환한다.

이번 주 정리

  • 음식 후보, 위치 정보, confidence, review flag를 분리하는 흐름 정리
  • 100g 기준 영양값과 사용자 입력량 기반 재계산 방향 정리
  • AI Hub와 실사용 사진을 함께 쓰는 라벨링 검증 계획 정리
YOLO / GCV 후보 100g Nutrition Roboflow 100장 검증

02
영양제 입력

영양제 라벨은 OCR 결과를 성분명·함량·단위 후보로 구조화하고, 주의 후보는 확정 판단이 아닌 검토 대상으로 분리한다.

이번 주 정리

  • 라벨 OCR 결과를 preview 상태로 유지하는 분석 흐름 정리
  • Google Vision, PaddleOCR, LLM 후처리 후보 비교 축 정리
  • 사용자 확인 전 복용 기록과 Agent context로 넘기지 않는 기준 정리
Google Vision PaddleOCR Review Gate

03
공통 기반

로그인, 프로필, 동의 상태는 음식·영양제 입력을 사용자 단위로 안전하게 연결하기 위한 기반이다. 이는 복약 주의 기능 완료가 아니라, 확인된 입력만 저장하고 이후 Agent 설명에 연결하기 위한 전제 조건이다.

정리된 기준

  • 사용자별 기록 분리와 refresh token 기반 인증 흐름
  • 만성질환, 복용 약, 목표 정보를 안전한 context로 관리
  • 외부 OCR 전송, 민감 건강 정보 활용, 사용자 확인 이력 분리

04
안전 기준

OCR/AI 결과는 후보이며, 건강 기록으로 확정되기 전 사용자의 확인 단계를 거쳐야 한다. 주의 문구는 현재 입력 정보 기준의 정보 제공으로 제한한다.

필수 원칙

  • preview 상태에서는 저장, 알림, Agent action side effect 차단
  • raw OCR text와 provider payload는 기본 저장 대상에서 제외
  • 영양·복약 주의 표현은 의학적 확정처럼 보이지 않게 검수
