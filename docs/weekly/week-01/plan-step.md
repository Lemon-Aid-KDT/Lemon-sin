## 1. Lemon Aid 프로젝트 개요

Lemon Aid는 경북대학교 AI/빅데이터 전문가 양성 과정에서 레몬헬스케어 협업 과제를 수행하는 팀이다. 본 프로젝트의 목표는 레몬헬스케어가 제시한 영양제 분석과 식단·활동 분석 과제를 바탕으로, 8주 안에 시연 가능한 AI 건강관리 서비스 MVP를 기획하고 구현하는 것이다.

우리가 제안하는 서비스는 **Lemon Aid Agent**다. 음식과 영양제를 사진 또는 텍스트로 기록하면, AI가 사용자의 기본 정보, 활동 정보, 만성질환·복약 정보를 함께 고려해 건강관리 참고 정보를 제공하는 모바일 기반 건강관리 보조 서비스다.

Lemon Aid Agent는 진단, 치료, 처방을 제공하는 의료 서비스가 아니다. 공식 영양 기준과 사용자 입력 데이터를 바탕으로 사용자가 자신의 식단·영양제·활동 상태를 이해하고, 필요한 경우 전문가 상담으로 이어질 수 있도록 돕는 서비스다.

## 2. 레몬헬스케어 과제 이해

레몬헬스케어가 제시한 과제는 크게 두 가지다.

첫째, **복용 중인 영양제 기반 분석**이다. 사용자의 성별, 나이, 키, 몸무게, 체형과 복용 중인 영양제를 바탕으로 개인별 섭취 영양소, 권장 섭취 기준, 부족 영양소, 섭취량 권고, 목적별 특화 분석을 제공하는 과제다.

둘째, **음식·식단·활동 정보 기반 분석**이다. 사용자의 식단, 성별, 나이, 키, 몸무게, 활동 정보, 걸음수를 바탕으로 섭취 영양소, 권장 기준, 체중 변화 예측, 활동 권고, 부족 영양소 추천을 제공하는 과제다.

두 과제의 공통점은 단순 정보 제공이 아니라 **사용자 개인의 생활 데이터와 건강 데이터를 연결해 맞춤형 건강관리 결과를 만드는 것**이다. 영양제 분석은 보충 섭취 상태를 보여주고, 식단 분석은 일상 섭취 상태를 보여주며, 활동 정보는 생활 습관과 체중 변화를 해석하는 기준이 된다.

## 3. Lemon Aid의 과제 해석

Lemon Aid는 이 과제를 영양제 앱이나 음식 인식 앱을 따로 만드는 일로 해석하지 않았다. 두 과제는 모두 레몬헬스케어가 보유한 의료 데이터 인프라와 연결될 때 더 큰 의미를 갖는다.

레몬헬스케어는 건강의신, LDB(Lemon Digital Bridge), 의료기관 네트워크, 표준화 의료데이터 API를 통해 병원성 데이터와 사용자 건강 데이터를 연결할 수 있는 기반을 갖고 있다. 따라서 본 과제는 단순 분석 기능 개발이 아니라, 병원 기록과 생활 기록을 함께 이해하는 **AI 건강관리 Agent 서비스의 가능성을 실증하는 과제**로 볼 수 있다.

Lemon Aid는 음식과 영양제 분석을 출발점으로 삼고, 여기에 만성질환, 복약 정보, 활동량, 체중 변화, 검사값을 연결하는 방향으로 과제를 해석했다. 이때 분석 알고리즘은 수치를 계산하고, Agent는 그 수치를 사용자가 이해할 수 있는 관리 언어로 바꾼다. 즉, 최종 방향은 **만성질환자와 예방 관리 사용자를 위한 개인화 건강관리 보조 서비스**다.

## 4. 해결하고자 한 문제

만성질환자와 건강관리가 필요한 사용자는 여러 정보를 따로 관리한다. 음식은 식단 앱이나 메모에, 영양제는 제품 사진이나 기억에, 복약은 알림 앱에, 활동량은 헬스 앱에, 검사값은 병원 기록에 흩어져 있다. 이 상태에서는 하루 전체 건강관리 흐름을 한 번에 이해하기 어렵다.

Lemon Aid가 해결하고자 한 문제는 다음과 같다.

| 문제 | 설명 |
|------|------|
| 데이터 분산 | 음식, 영양제, 복약, 활동량, 건강검진 정보가 서로 다른 곳에 흩어져 있다. |
| 통합 섭취량 파악 어려움 | 음식과 영양제를 따로 관리해 하루 전체 영양소 섭취량을 알기 어렵다. |
| 중복·과다 가능성 확인 어려움 | 여러 영양제를 함께 복용할 때 같은 성분이 중복되는지 확인하기 어렵다. |
| 개인화 부족 | 일반 권장량이나 칼로리 정보만으로는 만성질환, 복약, 건강 목적을 반영하기 어렵다. |
| 실천 방향 부족 | 정보는 많지만 오늘 무엇을 줄이고 무엇을 보완해야 하는지 이해하기 어렵다. |

이 문제를 해결하기 위해서는 음식과 영양제를 각각 분석하는 데서 끝나지 않고, 사용자의 건강관리 맥락 안에서 두 데이터를 함께 해석해야 한다.

## 5. 과제 해결 전략

Lemon Aid는 과제를 **입력 → 구조화 → 기준 비교 → 개인화 → 결과 제공 → 사용자 승인** 흐름으로 해결한다.

영양제 분석은 사용자가 복용 중인 영양제를 입력하거나 라벨을 촬영하는 것에서 시작한다. OCR과 LLM 구조화를 통해 제품명, 성분명, 함량, 단위를 추출하고, 식약처 건강기능식품 원료 DB와 공식 기준을 바탕으로 섭취 상태를 비교한다. 이후 부족하거나 과다할 가능성이 있는 영양소와 눈 건강, 간 기능, 피로 회복 등 목적별 분석 결과를 제공한다.

식단 분석은 음식 사진 또는 텍스트 식단 입력에서 시작한다. 음식 후보를 생성하고 공식 영양 데이터와 연결해 영양소를 산출한다. MVP 단계에서는 사진만으로 섭취량을 단정하지 않고 100g 기준 영양정보를 우선 제공하며, 사용자가 g 또는 인분을 입력한 경우 섭취량 기반 계산으로 확장한다. 이후 사용자 정보와 활동 정보를 결합해 체중 변화와 활동 관리 방향을 제공한다.

모든 AI 분석 결과는 자동 확정하지 않는다. 사용자가 미리보기에서 결과를 확인하고 수정한 뒤 승인해야 저장된다. 이는 AI 오류 가능성을 줄이고, 건강 정보에 대한 사용자 주도권을 보장하기 위한 원칙이다.

## 6. Lemon Aid Agent 서비스 개요

서비스명은 **Lemon Aid Agent**다.

> 사진 한 장으로 음식과 영양제를 기록하고, AI가 사용자의 건강 상태에 맞춰 영양·활동 관리 방향을 알려주는 건강관리 보조 서비스

Lemon Aid Agent의 핵심 사용자는 특정 연령대가 아니라 **만성질환 관리 또는 예방적 건강관리가 필요한 사용자**다.

| 사용자 유형 | 필요한 가치 |
|-------------|-------------|
| 만성질환 관리 사용자 | 식단, 영양제, 복약 정보를 함께 고려한 관리 방향 |
| 복약·영양제 병행 사용자 | 성분 합산, 중복 가능성, 전문가 상담이 필요한 지점 안내 |
| 예방 관리 사용자 | 음식·활동 기록 기반의 실천 피드백 |

이 서비스는 사용자의 건강 상태를 진단하지 않는다. 대신 입력된 음식·영양제·활동 정보를 정리하고, 권장 기준 대비 낮거나 높은 지점, 주의가 필요할 수 있는 지점, 전문가 상담이 필요한 지점을 이해하기 쉽게 안내한다.

Agent가 필요한 이유는 계산 결과를 사용자 맥락으로 바꾸기 위해서다. 알고리즘이 수치와 규칙을 담당한다면, Agent는 최근 식단·영양제·활동·복약 기록을 요약 기억으로 쌓아 두고 설명의 깊이를 개인화한다. 기록이 쌓일수록 같은 사용자에게 더 맞는 관리 방향을 제안할 수 있다.

## 7. 핵심 기능

Lemon Aid Agent의 핵심 기능은 레몬헬스케어 과제의 출력 항목과 직접 연결된다.

| 기능 | 사용자에게 보이는 가치 | 과제 연결 |
|------|------------------------|-----------|
| 영양제 라벨 분석 | 성분명, 함량, 단위를 자동 정리한다. | 영양제 분석 |
| 음식 사진·텍스트 식단 분석 | 식단을 영양소 단위로 변환한다. | 식단 분석 |
| 음식+영양제 통합 합산 | 하루 전체 섭취 영양소를 함께 본다. | 부족 영양소, 권장 섭취량 |
| 부족·과다 가능성 안내 | 권장 기준 대비 낮거나 높은 영양소를 확인한다. | 부족 영양소 추천 |
| 체중 변화 예측 | 섭취와 활동 정보를 바탕으로 변화 가능성을 본다. | 체중 예측 |
| 활동 권고 | 걸음수와 활동 정보를 건강관리 방향으로 연결한다. | 활동 권고 |
| 목적별 분석 | 눈 건강, 간 기능, 피로 회복 등 목적에 맞춰 결과를 해석한다. | 목적별 특화 분석 |
| 챗봇 설명·알림 | 분석 결과를 쉽게 묻고, 복약·식단 알림으로 연결한다. | 지속 관리 |

## 8. 서비스 구현 방식

Lemon Aid Agent는 모바일 앱, 백엔드 API, 분석 알고리즘, AI Agent가 연결된 구조로 구현한다.

**모바일 앱**은 Flutter 기반으로 음식·영양제 촬영, 분석 결과 대시보드, 챗봇 화면, 알림·기록 흐름을 제공한다. 사용자는 사진 또는 텍스트로 데이터를 입력하고, 분석 결과를 미리보기에서 확인한 뒤 저장한다.

**백엔드 API**는 FastAPI 기반으로 인증, 입력 검증, 사진 업로드, 분석 요청, 데이터 저장, Agent 호출을 처리한다. 앱은 DB에 직접 접근하지 않고 API를 통해서만 데이터를 주고받는다.

**분석 알고리즘**은 OCR, 라벨링, CSV/API 매칭, KDRIs 기준 비교, BMI·BMR·TDEE·체중 예측, 활동점수 계산을 담당한다. 결정 가능한 계산은 알고리즘으로 처리하고, LLM은 비정형 텍스트 구조화와 설명 생성을 보조한다.

**AI Agent**는 3개 역할로 나뉜다.

| Agent | 역할 |
|-------|------|
| 개인화 Agent | 만성질환, 복약, 검사값, 건강 목적을 결과 해석 기준으로 변환한다. |
| 평가 Agent | 부족·과다 가능성, 식단관리 점수, 개선 방향을 생성한다. |
| 챗봇 Agent | 분석 결과를 쉬운 말로 설명하고 알림·캘린더 등록을 보조한다. |

안전 장치도 함께 적용한다. AI 응답은 Pydantic 스키마 검증, 금지 표현 필터, 사용자 미리보기, 승인 후 저장 흐름을 거친다. 의료적 단정 표현은 사용하지 않고, 복약이나 영양제 상호작용 가능성은 전문가 상담 권장으로 안내한다. Agent는 저장된 전체 원본을 그대로 외우는 방식이 아니라, 사용자 맥락을 요약 기억으로 관리해 반복 사용할수록 개인화 품질이 높아지게 설계한다.

## 9. 레몬헬스케어 목표와의 연결

레몬헬스케어의 궁극적인 강점은 병원성 데이터와 사용자 건강 데이터를 연결할 수 있는 인프라에 있다. 건강의신, LDB, 의료기관 네트워크, 표준화 의료데이터 API는 사용자의 병원 기록, 검사값, 복약 정보와 생활 데이터를 연결할 수 있는 기반이다.

Lemon Aid Agent는 MVP 단계에서 건강의신과 직접 연동하지 않는다. 대신 독립형 참조 앱으로 구현해, 향후 건강의신 또는 LDB/FHIR 기반 연계 시 어떤 사용자 경험과 데이터 흐름이 가능한지 보여주는 역할을 한다. 즉, 레몬헬스케어가 가진 병원성 데이터 인프라와 사용자의 생활 데이터를 연결할 때 어떤 형태의 개인화 건강관리 서비스가 가능한지 보여주는 출발점이다.

음식과 영양제 분석은 사용자의 생활 데이터를 만드는 진입점이다. 여기에 검사값, 만성질환 태그, 복약 정보를 연결하면 단순 칼로리 앱이나 영양제 정보 앱을 넘어, 레몬헬스케어가 지향하는 **개인 건강 데이터 기반 맞춤형 건강관리 서비스**로 확장할 수 있다.

따라서 Lemon Aid는 레몬헬스케어의 의료 데이터 인프라가 생활 건강 데이터와 만났을 때 어떤 서비스를 만들 수 있는지 보여주는 실증형 MVP로 해석할 수 있다. 건강의신과 LDB는 이 서비스가 향후 확장될 때 연결될 수 있는 목표점이며, 우리는 그 가능성을 독립형 참조 앱에서 먼저 검증한다.

| 사용자 | 흩어진 음식·영양제·활동 정보를 한 흐름으로 이해하고, 오늘의 실천 방향을 확인할 수 있다. |
| 레몬헬스케어 | 건강의신 확장 가능성, LDB 기반 개인화 건강관리 UX, AI Agent형 서비스 모델을 검토할 수 있다. |
| Lemon Aid 팀 | 8주 안에 시연 가능한 MVP를 구현하고, 과제 수행 결과를 서비스 모델로 확장해 제안할 수 있다. |

## 11. 안전 및 한계

Lemon Aid Agent는 의료 서비스를 대체하지 않는다. 분석 결과는 사용자가 건강관리를 이해하기 위한 참고 정보이며, 질환 진단, 치료, 처방, 복용량 변경 지시로 표현하지 않는다.

문서와 화면 문구는 다음 기준을 따른다.

- "결핍입니다" 대신 "권장량 대비 낮습니다" 또는 "낮을 가능성이 있습니다"로 표현한다.
- "위험합니다" 대신 "주의가 필요할 수 있습니다"로 표현한다.
- 복약이나 영양제 상호작용 가능성은 "전문가 상담을 권장합니다"로 안내한다.
- OCR과 AI 분석 결과는 틀릴 수 있으므로 사용자가 확인하고 수정할 수 있게 한다.
- 만성질환, 복약, 검사값 등 민감 건강정보는 최소 수집, 별도 동의, 안전한 저장 원칙을 따른다.

## 결론

Lemon Aid는 레몬헬스케어가 제시한 영양제 분석과 식단·활동 분석 과제를 먼저 수행 가능한 분석 흐름으로 나누었다. 그리고 두 과제를 단순 기능이 아니라 생활 데이터와 병원성 데이터를 연결하는 개인화 건강관리 서비스의 출발점으로 해석했다. 이 해석의 중심에는 Agent가 있으며, 반복 기록이 쌓일수록 더 사용자에게 맞는 설명과 관리 방향을 제공하게 된다.

그 결과 제안하는 Lemon Aid Agent는 음식과 영양제를 쉽게 기록하고, 사용자의 건강관리 맥락에 맞춰 부족·과다 가능성, 체중 변화, 활동 방향, 목적별 분석을 제공하는 AI 건강관리 보조 서비스다. MVP는 독립형 참조 앱으로 구현하되, 향후 레몬헬스케어의 건강의신·LDB 기반 서비스 확장 가능성을 고려한다.
