> 확인일: 2026-05-14  
> 성격: 국내 학회 진료지침, 질환 관리 가이드, 복약 안전 안내  
> 사용 범위: 팀 학습, 멘토·의료자문위 질문 정리, 안전선 검토

## 핵심 판단

국내 임상 가이드라인은 질환을 이해하는 데 필요하지만, Lemon Aid가 진료지침의 치료 목표나 처방 기준을 앱 로직으로 자동 적용하면 안 됩니다. 이 태그의 자료는 의료자문위 검토 전에는 사용자별 권고 규칙으로 사용하지 않습니다.

## 사용할 수 있는 방식

- 팀 학습: 질환별로 왜 특정 영양소·생활습관을 조심스럽게 봐야 하는지 이해
- 전문가 검토: 의료자문위에 질환별 보정, 금지 표현, 상담 권장 조건을 질문할 때 근거로 사용
- 제품 안전선: “앱이 하면 안 되는 판단”을 식별하는 참고 자료로 사용
- 문구 검수: 치료·처방처럼 보이는 표현을 피하고 “확인”, “주의 가능성”, “상담 권장”으로 낮춤

## 사용하면 안 되는 방식

- 혈당, 혈압, 지질, 골밀도 기준으로 앱이 질환 상태를 판정하지 않는다.
- 진료지침의 약물 치료 기준을 챗봇 답변이나 알림 로직에 넣지 않는다.
- 질환별 식단 처방표를 자동 추천 알고리즘으로 만들지 않는다.
- 사용자의 검사값만으로 치료 목표 달성 여부를 확정하지 않는다.
- 특정 질환자에게 영양제 복용량을 제안하지 않는다.

## 반영 위치

| 영역 | 반영 방식 |
|------|-----------|
| DB | 문헌 메타데이터만 저장, 치료 기준값은 저장하지 않음 |
| 알고리즘 | 의료자문위 검토 전 직접 반영 금지 |
| Agent | 질환 배경 설명과 상담 권장 문구 수준으로 제한 |
| UI/UX | 질환별 “확인해 볼 항목” 표현 검토 |
| 안전/검증 | 금지 표현, 상담 권장 조건, 멘토 질문 목록 |

## MVP 반영

- 사용자 온보딩에서 질환명은 배경 태그로만 저장한다.
- 평가 결과는 “질환 개선”이 아니라 “식단·영양소 섭취 상태 확인”으로 표현한다.
- 당뇨병, 고혈압, 이상지질혈증, 골다공증, 빈혈은 팀 학습 문서와 안전 문구 검토에 연결한다.

## v2 이후 검토

- 의료자문위 승인 후 질환별 “주의해서 볼 지표” 목록을 제한적으로 구조화
- 복약 정보와 영양제 성분의 상담 권장 룰을 약사 검토 후 도입
- 검사값 기반 표현은 법무·의료 검토 후 별도 기능으로 분리
