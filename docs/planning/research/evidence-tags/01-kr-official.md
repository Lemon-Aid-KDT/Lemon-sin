# KR_OFFICIAL: 구현 기준 가능

> 확인일: 2026-05-14  
> 성격: 국내 공식 기준, 공공 데이터, 공공 건강정보  
> 사용 범위: DB 기준값, 알고리즘 룩업, 앱 내 출처 표시, 안전 문구의 1차 기준

## 핵심 판단

`KR_OFFICIAL`은 Lemon Aid에서 실제 구현 기준으로 사용할 수 있는 가장 높은 우선순위 자료입니다. 다만 공식 자료라도 질병의 진단, 치료, 처방을 앱이 수행하는 근거로 확장하면 안 됩니다.

## 사용할 수 있는 방식

- DB: 식품명, 식품군, 영양소, 1회 제공량, 출처 코드, 영양소 기준값, 상한섭취량 저장
- 알고리즘: 사용자의 나이·성별 기준으로 KDRIs 룩업, 식단+영양제 섭취량 합산, 권장량 대비 비율 계산
- Agent: 공식 기준 계산 결과를 쉬운 말로 설명하고, 불확실한 입력은 사용자 확인을 요청
- UI/UX: “권장량 대비”, “상한섭취량 초과 가능성”, “섭취 시 주의사항 확인”처럼 참고형 문구 사용
- 안전 문구: 질병관리청·식품안전나라 표현을 바탕으로 진단·치료 단정 표현을 피함

## 사용하면 안 되는 방식

- 공식 자료의 질환 설명을 사용자 진단 결과처럼 표시하지 않는다.
- KDRIs 기준을 만성질환자 개별 치료 목표로 바꾸지 않는다.
- 건강기능식품 기능성 문구를 질병 예방·치료 효과처럼 확장하지 않는다.
- DUR 또는 복약 주의 자료를 앱이 병용 가능/불가능을 최종 판정하는 근거로 쓰지 않는다.
- 식품 DB의 평균 영양성분을 사용자의 실제 섭취량 확정값처럼 표시하지 않는다.

## 반영 위치

| 영역 | 반영 방식 |
|------|-----------|
| DB | `foods`, `food_nutrients`, `supplement_ingredients`, `nutrient_reference`, `evidence_sources` |
| 알고리즘 | KDRIs 대비 비율, 상한섭취량 초과 가능성, 식품 DB 매칭 |
| Agent | 공식 기준 기반 설명, 불확실성 안내, 전문가 상담 권장 |
| UI/UX | 출처 표시, 권장량 대비 그래프, 사용자 수정/승인 흐름 |
| 안전/검증 | 금지 표현 필터, 공식 출처 표시, 기준 버전 관리 |

## MVP 반영

- KDRIs 2020을 1차 기준으로 사용하고, 정오표 반영 여부를 별도 체크한다.
- 음식 영양성분은 식약처 DB와 농진청 DB를 우선 매칭한다.
- 건강기능식품은 제품 추천이 아니라 성분·함량·기능성·주의사항 표시 중심으로 다룬다.
- 질환 정보는 사용자의 배경 맥락으로만 사용하고, 질환별 식단 처방은 제외한다.
