# LEMON-AID — 비전공자용 설명

## 한 줄로 말하면

**음식 사진 찍으면 그게 뭔지 알아보고 칼로리·영양 정보를 알려주는 앱**을 만들고 있다.

---

## 어떻게 작동하나

사용자가 식당에서 비빔밥 사진을 찍는다 →
앱이 "이거 비빔밥이네, 한 그릇 559 kcal" 라고 말해준다 →
필요하면 "비빔밥인데 어떤 종류인가?" (전주식 / 돌솥 / 일반) 도 물어본다.

음식 종류는 **316가지** 인식 가능. (한식 위주, 디저트 추가 예정)

---

## AI 직원 두 명이 협동

비유하자면 **사진을 두 명한테 보여주는 것**과 같다.

**1번 직원 — "찾는 사람" (Detector)**
- 사진에서 음식이 "어디에" 있는지만 찾는다
- "여기 동그란 그릇 안에 뭔가 있다" 정도만 본다
- 음식인지 아닌지, 위치만 찍어서 잘라낸다

**2번 직원 — "아는 사람" (Classifier)**
- 1번이 잘라준 음식 부분만 본다
- "이거 비빔밥이다", "이거 김치찌개다" 같이 정확한 음식 이름을 맞춘다
- 316개 음식 중 어느 거인지 골라낸다

마지막에 **요리 사전** (영양 정보 DB) 에서 그 음식의 칼로리·영양소를 찾아서 알려준다.

---

## 왜 두 명으로 나눴나

한 명한테 "어디 있고 + 그게 뭔지" 둘 다 시키면 어렵고 느림. 둘로 나누면:
- 각자 한 가지만 잘하면 됨 (정확도 ↑)
- 더 가벼워서 휴대폰에서 빨리 돌아감 (속도 ↑)

---

## 데이터는 어디서

5가지 소스를 합쳤다:

| 출처 | 무엇 |
|---|---|
| AI Hub 한국 음식 데이터 | 316종 음식 사진 50만 장 (가장 핵심) |
| 식약처 영양 DB | 19,495개 음식 영양 정보 |
| Gemini 라벨링 | AI 가 직접 찍어준 박스 1만 2천 개 |
| Florence-2 자동 박스 | 또 다른 AI 가 만든 박스 1만 2천 개 |
| 비식품 사진 (Kaggle/Food-5K) | "이건 음식이 아니다" 가르치는 용도 |

## 지금 어디까지 왔나 (2026-05-27 기준)

```
[데이터 정리]  →  [학습 중]      →  [평가]    →  [모바일 이식]  →  [출시]
   ✓완료          현재 위치          대기         대기            대기
```

**진행 중**:
- A100 큰 컴퓨터에서 "아는 사람" (Classifier) 학습 중
- 노트북에서 "찾는 사람" (Detector) 학습 중
- 두 학습이 끝나야 평가 단계로 갈 수 있음

**완료된 것**:
- 데이터 5종을 한 곳에 모음
- 음식 이름 정리 (애매한 이름 3개 발견 - 불고기 등)
- 데이터 통합 규칙 문서화 (앞으로 새 데이터 들어와도 같은 규칙으로 처리)
- 평가 기준 문서화 (어떤 숫자를 봐야 잘 되는 건지)

## 앞으로 할 일

| 단계 | 무엇을 하나 | 왜 |
|---|---|---|
| 1. 학습 완료 | 두 AI 직원 가르치기 끝내기 | 모델 확보 |
| 2. 실사진 평가 | 진짜 핸드폰 사진으로 시험 | 실전에서도 맞추나? |
| 3. 모델 다이어트 | 모델 크기 1/4 로 줄이기 (양자화) | 휴대폰에 들어가게 |
| 4. 앱 이식 | Android / iOS 앱에 넣기 | 사용자가 쓸 수 있게 |

---

## 왜 어려운가

1. **음식은 비슷한 게 많음** — 김치찌개 vs 부대찌개, 라면 vs 라멘
2. **사진마다 다름** — 같은 비빔밥도 식당마다 모양 다름
3. **크기 제약** — 휴대폰에 들어갈 만큼 가벼워야 함 (그러면서 정확도 유지)
4. **속도 제약** — 사진 찍으면 1초 안에 답이 나와야 함

---

## 핵심 약속 (절대 어기지 않을 규칙)

1. 사람 눈으로 구분 가능한 음식만 따로 가르친다 (안 되면 변형으로 처리)
2. 식약처 DB 가 영양 정보의 유일한 기준이다
3. 새 데이터는 정해진 절차로만 들어온다 (즉흥적으로 추가 금지)
4. 모든 변경은 기록한다 (잘못되면 되돌릴 수 있게)
