# 박스 어노테이션 가이드 — Food Region (1-class)

> Detector 학습/평가의 **모든** 박스는 이 기준 하나로 통일한다.
> train(296·v3)·val·296 test·Real App Test 전부 동일 기준. 안 그러면 mAP가 무의미해진다.
> 원칙: **recall 우선** — 애매하면 "친다" 쪽. 단 명백한 비음식·배경은 안 친다.

---

## 1. 기본 규칙 — ★ "단위는 그릇, 영역은 음식"
- **몇 개 칠지 = 그릇/접시 단위.** 한 그릇에 담긴 음식 = 1박스 (한상이면 그릇 수만큼).
- **어디까지 칠지 = 음식이 담긴 그릇/접시 포함.** (296 라벨 실측 결과 그릇 포함 기준 — 학습·평가 일관성을 위해 그에 맞춤) 단 식판·쟁반·상 전체 포함은 X.
- 같은 음식 여러 조각 한 접시(만두 5개) = 1박스. / 한 접시에 다른 음식 구분 담김(도시락 칸·모듬) = 음식별.
- **식판/쟁반/상 전체 1박스 = 절대 금지** (우산박스).
- 근거: 학습 주력인 296이 "한 그릇 음식=1박스" 단위라, 전 데이터·테스트가 같은 자로 재야 mAP가 유효함.
- 클래스는 항상 **0 (food)**. 종류 구분 안 함.
- 좌표는 보이는 음식 영역 기준 (가려진/잘린 부분 추정 금지 — 보이는 만큼만).

## 2. Edge case 확정 기준
| 상황 | 기준 |
|---|---|
| **모듬접시** (한 접시 여러 음식) | 시각적으로 분리되면 음식별 박스. 섞여 구분 안 되면 접시 단위 1박스. |
| **국/탕/찌개** | **그릇 포함** 1박스 (국물+건더기+그릇). 받침/쟁반 제외. |
| **같은 음식 여러 개** (만두·꼬치·전) | 한 접시에 모여있으면 **접시 단위 1박스** (개별 카운팅 X). 흩어져 있으면 묶음별. |
| **밥·공기밥** | 1박스. |
| **양념·소스 종지** (간장·쌈장·초장) | **친다** (먹는 것). 단 **빈 종지·물·간장만 담긴 빈컵**은 X. |
| **음료 (컵·잔)** | 내용물 있으면 **컵 전체 1박스**. **빈 컵·빈 종이컵**은 X. |
| **반쯤 가림 / 프레임 잘림** | 음식으로 인식되면 **보이는 부분에 박스** (가림 50%+ 여도 침). |
| **배경 음식** (멀리 다른 테이블) | 주 피사체 아니면 **제외** (애매하면 제외 — 노이즈 방지). |
| **너무 작음** | 이미지 면적 **~1% 미만** 자잘한 건 스킵. |

## 3. 박스 안 치는 것 (비음식)
- 냅킨·수저·젓가락·포크, 손·팔, 휴대폰, 포장지·비닐, 메뉴판, 빈 그릇/빈 컵, 식탁 무늬, 물티슈.
- → 이런 게 사진에 있고 음식이 없으면 **빈 라벨(background)** 이미지로 둔다 (FP 억제).

## 4. 소스별 적용 메모
- **296**: 단일음식 1박스 = 이미 이 기준에 부합 (그대로 사용).
- **v3 orig/rb**: 항목별 박스 — 기준 대체로 일치. 어긋난 것만 수정.
- **v3 fl (Florence 자동)**: 과분할(자잘·중첩) 많음 → 정제 시 "접시 단위·타이트" 기준으로 거름.
- **Real App Test**: 이 가이드대로 사람이 직접 라벨.

## 5. 일관성 체크 (라벨 후)
- 같은 음식인데 어떤 데서는 접시단위, 어떤 데선 항목별로 친 케이스 없는지 샘플 감사.
- 빈 컵/종이컵을 친 박스 없는지 (있으면 제거 — not-food).
- 박스 최소 크기(<1%) 위반 없는지.
