# docs/ — Lemon Aid 문서 묶음

`PROJECT_GUIDE.md`에 모두 넣기에는 큰 문서와 작업·검토용 분리 문서를 목적별로 관리합니다.
새 문서를 만들 때는 [문서 라우팅 하네스](./harness/document-routing.md)를 먼저 확인합니다.
Codex 실행 규칙은 repo 루트의 `AGENTS.md`, `.codex/`, `.agents/skills/`를 기준으로 합니다.

## 문서 카테고리

| 폴더 | 성격 |
|------|------|
| [guide/](./guide/) | 제품·개발 기준 가이드 |
| [domain/](./domain/) | 만성질환, 영양, 식단 등 팀 내부 도메인 학습 자료 |
| [research/](./research/) | 논문, API, 외부 자료, 기술 판단 근거 |
| [reports/](./reports/) | 검토 보고서, 리뷰 결과, 의사결정 기록 |
| [appendices/](./appendices/) | 본문 가이드에서 분리한 부록성 실행 자료 |
| [presentations/](./presentations/) | 멘토 미팅, 발표, PDF/HTML 산출물 |
| [harness/](./harness/) | 문서 자동 분류 규칙과 템플릿 |

## 새 문서 작성 규칙

- 문서 목적을 기준으로 먼저 기존 카테고리에 배치합니다.
- 맞는 성격이 없으면 새 폴더명을 제안하고 승인 후 생성합니다.
- 새 문서를 추가하거나 이동하면 이 인덱스와 해당 폴더의 `README.md`를 함께 갱신합니다.
