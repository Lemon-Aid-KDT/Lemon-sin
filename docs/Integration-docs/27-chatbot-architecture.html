사용자 맥락을 모아 개인화하고, 검수된 지식을 RAG로 검색해 근거 있는 답을 만드는 흐름

건강·식단·복약·영양제에 대한 자연어 질문
질문
Lemon Aid 챗봇 — 개인화 컨텍스트 + 근거 검색 오케스트레이터
1🧠 개인화 컨텍스트 수집 — UserHealthContextSnapshot
프로필·질환·약·영양제·최근 식사·오늘 분석을 한 객체로 (확인된 데이터만, raw 제외)
2🔍 ContextResolver — 필요한 맥락만 선별
질문마다 관련 맥락만 선택 (부족하면 추가 조회 / 정보 부족이면 되묻기)
3📚 RAG 근거 검색 — 검수 지식 검색
의도·엔티티 분류 → 키워드+벡터 하이브리드 검색 → 검수·유효·노출허용 필터
4🗂️ AnswerCardNormalizer → AnswerCard
검수 근거를 답변 프레임으로 정리하고 개인화 맥락과 결합
5🤖 LLM 표현 (판단자 아님)
AnswerCard → 자연스러운 한국어 답변 (근거 밖 사실 생성 금지)
⚙️ 결정론적 렌더러
경계·모름·교체 응답은 LLM 없이 규칙으로 생성
6🔒 안전 점검 (SafetyGuard)
금지 문구·근거 없는 사실/수치 차단 → 문제 시 결정론적 답변으로 교체
개인화 + 검수 근거
💬 응답 (답변 + answerability + sources[])
개인화된 답변과 함께 답변 상태·검수 출처를 같이 전달

★ 개인화 ① 무엇을 가져오나 — UserHealthContextSnapshot

답변 직전, 사용자의 건강 데이터를 하나의 안전한 객체로 모읍니다. 이 맥락이 있어야 "일반론"이 아니라 "이 사용자에게 맞는" 답이 나옵니다.

👤 프로필 · 질환

키·몸무게·만성질환·복약·목표. 질환이 있으면 건강 축이 자동 확장됩니다 (예: 고혈압 → 나트륨·혈압 축).

📈 오늘 / 전체 분석

오늘 식사·영양제·체크리스트 상태와 누적 건강 분석 스냅샷.

💊 활성 영양제

등록된 영양제와 오늘 체크 여부 (병용 주의 판단의 입력).

🍽️ 최근 식사 · 체크리스트

최근 식사 기록(최근 10개)과 체크리스트 — 확인된 기록만 포함.

💬 대화 건강 신호

대화에서 추출한 건강 신호 (예: 어지러움·증상 언급). 후속 질문 맥락 유지에 사용.

🪞 사용자가 본 분석

사용자가 화면에서 본 분석 맥락 — 오래된(stale) 정보 기준 답변을 막기 위한 참조.
불변식: raw OCR·프롬프트·LLM 출력은 스냅샷에서 자동 제거되고, 사용자가 확인(confirmed)한 데이터만 들어갑니다. 그래서 개인화하면서도 민감정보·미확정 정보가 새지 않습니다.

★ 개인화 ② 필요한 맥락만 — ContextResolver

스냅샷을 통째로 넣지 않습니다. 질문 유형에 따라 필요한 맥락만 선택하고, 모자라면 추가로 가져오거나 되묻습니다.

✅ 충분

스냅샷만으로 답할 수 있음 → 그대로 사용.

🔎 추가 조회 필요

needs_more_info

❓ 정보 부족

맥락이 비어 있으면 답을 만들지 않고 필요한 정보를 되묻기.
왜 선별하나: 매 요청에 모든 기록을 주입하면 토큰 낭비와 노이즈가 생깁니다. 질문과 관련된 맥락만 골라야 답이 정확하고 빨라집니다.

🔎 RAG 근거 검색 — 어떤 정보를, 어떻게 가져오나

개인화 맥락이 정해지면, 검수된 의료·영양 지식을 RAG로 검색해 답의 "사실 근거"를 가져옵니다.

① 어떻게 — 검색 흐름

질문 의도·엔티티 분류
→
하이브리드 검색
(키워드 + 벡터 임베딩)
→
순위 결합 · 안전/신뢰 reranking
→
governance 필터
(검수 · 유효 · 노출허용)
→
AnswerCard 정규화

② 어떤 정보 — 검수 지식에 담긴 것

검색은 "아무 문서"가 아니라, 검토를 거친 지식의 정리된 항목을 가져옵니다.

🗣️ 허용 문구

사용자에게 말해도 되는 표현·원리.

🥗 구체 예시

음식·행동 후보 (예: 나트륨↓ 식품).

📋 오늘 점검 항목

지금 확인하면 좋은 것들.

⚠️ 주의 조건

전문가 상담이 필요한 시점.

🚫 금지 표현

이 주제에서 말하면 안 되는 표현.

🔖 출처

발행처·버전·검토일·유효기간.
핵심: 검색 결과 원문(raw)을 LLM이나 사용자에게 그대로 주지 않습니다. 항상 AnswerCard로 정규화해서, 검수·유효·노출 허용된 근거만 개인화 맥락과 함께 전달합니다.

🔒 안전·근거 원칙 (개인화·검색 위에 항상)

개인화와 검색이 아무리 풍부해도, 답은 검수된 지식 안에 머뭅니다. 근거 없음 검수 근거가 없으면 일반 지식으로 채우지 않고 "지금은 모른다"고 말합니다. 개인 의료 결정 진단·처방·용량·검사수치 해석은 결정하지 않고, 가져갈 정보를 안내합니다. 응급·고위험 응급·고위험 상호작용(P0)은 LLM 없이 결정론적으로 처리합니다. LLM은 끝까지 표현 보조자이며 판단자가 아닙니다.

📌 구성요소 역할 요약 (개념)

구성요소 핵심 역할 하지 않는 것
개인화 컨텍스트 수집
(Snapshot)		 프로필·질환·약·식사·분석을 한 객체로 모아 답을 개인화 raw OCR/프롬프트·미확정 데이터 포함
ContextResolver 질문별 필요한 맥락만 선택, 부족하면 추가 조회·되묻기 모든 기록을 항상 주입
RAG 근거 검색 키워드+벡터로 검수 지식을 검색해 사실 근거 확보 미검수·만료 자료를 답변에 사용
AnswerCard 정규화 검수 근거를 답변 프레임으로 정리하고 개인화 맥락과 결합 검색 원문(raw)을 그대로 전달
LLM 표현 AnswerCard를 자연스러운 한국어 문장으로 변환 근거 밖 의료 사실·수치 생성, 진단·처방 결정
결정론적 렌더러 경계·모름·교체 응답을 규칙으로 생성 응급·경계 판단을 LLM에 위임
안전 점검 금지 표현·근거 없는 사실/수치 차단 → 문제 시 fallback LLM 답변을 무검증으로 전달
