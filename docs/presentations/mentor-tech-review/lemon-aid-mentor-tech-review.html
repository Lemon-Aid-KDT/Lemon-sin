OCR/AI

OCR & AI BOUNDARY

AI는 건강 판단자가 아니라 구조화와 설명을 보조합니다.

Pipeline

OCR → LLM 구조화 → DB 매칭

  • Google Vision으로 라벨·메뉴 텍스트 추출
  • CLOVA OCR을 한국어 라벨 폴백으로 검토
  • Claude로 정해진 Pydantic JSON 구조화
  • OpenAI는 장애 시 백업 Provider
Agent Roles

분석은 알고리즘으로, 3개 Agent는 역할을 유지합니다.

분석 단계는 OCR·공식 DB 매칭·영양소 합산·결핍 판단 알고리즘으로 처리합니다.

개인화, 평가, 챗봇 Agent는 기준 반영, 결과 해석, 사용자 실행 보조를 담당합니다.

LLM 결과는 공식 DB·규칙·사용자 확인을 통과해야 사용자 결과가 됩니다.

EVIDENCE USE

자료사용 위치금지할 사용 방식
KDRIs권장 섭취량, 부족·과다 기준 비교질환별 치료 권고로 확장하지 않음
식약처 식품영양성분 DB음식 영양소 매칭출처 없는 음식 데이터로 대체하지 않음
건강기능식품 DB영양제 성분, 함량, 기능성 표현 확인특정 제품 추천으로 사용하지 않음
농진청 식품성분표식품 영양소 보강사진만으로 섭취량을 확정하지 않음
