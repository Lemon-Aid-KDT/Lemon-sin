# exp12 계획 — 진짜 약점 클래스 실데이터 보강 (PLAN ONLY)

> exp11(cap1500) 분석 결과 기반. **실행 전 계획 단계.** 2026-06-02

## 1. 배경 — exp11에서 확정된 것
- **과적합 = 데이터 부족** 확정: cap1500로 val 0.877→0.895(+0.018), train-val 갭 0.117→0.097.
- **부족클래스는 cap·복제·증강 다 안 먹힘** → 실데이터만 해법 (cap정체 +0.0007 / exp04·05 복제 -0.03 / AIHub 소진, 3증거).
- **🔑 핵심 재발견: "저데이터" ≠ "저성능".** 저데이터 12클래스 중 **8개는 AP≥0.89(이미 충분)**. 진짜 약한 건 4개뿐 → **전부 보강할 필요 없음.**

## 2. 진짜 보강 대상 (저데이터 AND 저AP <0.82)

| spicy-mixed-noodles (비빔국수) | 370 | 0.743 | selectstar 없음 → 신규수집/병합 |

## 3. exp12 설계 — takoyaki 단일 파일럿 (변수통제)
**가설**: 부족·약점 클래스는 실데이터로 개선된다 (복제와 달리).

- **Base**: taxo59 cap1500 (exp11과 동일 — 통제).
- **변수**: **takoyaki 한 클래스에만 selectstar 실데이터 추가**. (다른 모든 것 고정)
- **왜 takoyaki만?**: 4개 약점 중 **유일하게 selectstar로 즉시·검증 가능**. 입증되면 나머지는 신규수집으로 확대. 1변수라 인과 명확.
- **비교**: exp11 대비 **takoyaki AP 상승 + 타 클래스 비손상** → "실데이터가 약점 부족클래스 해법" 입증.

## 4. 데이터 파이프라인 (takoyaki 파일럿)
1. `selectstar/takoyaki/json` → food_class=takoyaki 확인 (매핑 정확)
2. exp10/11 best.pt로 박스 추론(conf 낮게) → **단, takoyaki 일치율 0%(OOD)였으므로 박스 신뢰 낮음** → **full-image-box fallback + 사람검수 필수**
3. YOLO 라벨(class=takoyaki idx, selectstar의 정답라벨 사용, 모델클래스 무시) 생성
4. taxo59 train에 추가 (takoyaki 410 → 목표 ~900~1000), **cap1500·val·기타 동일 유지**
5. 재학습 = exp12 (seed 42, 가능하면 seed 추가로 노이즈 분리)

## 5. 성공 기준
- ✅ takoyaki AP **0.77 → +0.05 이상** 상승 (동일 val 기준)
- ✅ 다른 58클래스 평균 비손상 (±0.01 내)
- 실패(안 오름/노이즈) → selectstar 스튜디오 도메인 한계 결론 → 신규수집(실환경)으로 전환

## 6. exp12 이후 — 나머지 약점 처리 로드맵
- **신규수집(web/CVAT)**: spicy-mixed-noodles, rice-soup (selectstar 없음, 실데이터 필요)
- **병합 검토**: tteokbokki-jajang(230·0.79) → tteokbokki-red 흡수? (자장떡볶이 희소). 비빔국수↔비빔면, 국밥↔국 인접 병합도 검토 (mala/탕수육/제육 drop 선례)
- **저AP-but-고데이터 클래스**: 데이터 문제 아님 → 라벨감사 or 택소노미(혼동군) 재점검

## 7. 리스크
- selectstar = **스튜디오 도메인**(AIHub와 동일) → 실환경(카톡) 갭은 못 닫음. takoyaki도 일치율 0%(OOD)라 박스/효과 제한적일 수 있음.
- 1클래스 파일럿 → 결과 노이즈 가능 (val ~100장). seed 2 권장.
- 박스 품질(모델 OOD) → full-image-box+검수로 방어. 멀티박스 이미지는 보류.

## 8. 병행/대안 (선택, 별도 실험)
- **exp13 후보**: 과적합 추가완화 — cap2000 또는 aug강화·weight_decay↑ (train 0.992 여전히 암기 → 풍부클래스 추가 개선 여지).
- selectstar 전체는 **학습보다 cross-domain 평가셋으로 동결** 권장 (이전 워크플로 결론) — takoyaki는 그 예외적 파일럿.
