# 한식 탐지 모델 실험 종합 정리 (exp06 → exp16)

> 작성 2026-06-08. 목표 = **실제 폰 사진에서 한식을 잘 맞추는 YOLO 탐지 모델**.
> 이 문서는 exp06부터 exp16까지의 흐름을 **쉽게** 정리한다.

---

## 0. 핵심 한 줄

**분류체계(클래스 설계)는 거의 다 짜냈고, 실제 사진 점수는 못 올렸다 → 진짜 해법은 "실데이터"임이 exp16에서 통계적으로 입증됐다.** 실데이터 단 2%(1,177장)를 섞었더니 실제 사진 인식률이 **+0.109(유의)** 올랐다.

---

## 1. 먼저 알아야 할 4가지 개념 (쉽게)

| 개념 | 쉬운 설명 |
|---|---|
| **studio 사진** | AIHub 데이터셋의 **깔끔한 음식 사진**(조명·구도 일정). 학습에 주로 씀. |
| **wild 사진** | 사용자가 실제 폰으로 찍은 **지저분한 실환경 사진**(카톡 등). 우리가 진짜 맞춰야 할 대상. |
| **mAP / 인식률** | 모델 점수. 단 **studio mAP는 "누수"로 부풀려진 가짜 점수**(아래) → 비교용으로만, 절대값 신뢰 금지. **wild 인식률이 진짜 실력.** |
| **누수(leakage) & 도메인갭** | AIHub는 같은 음식코드 사진이 학습·시험 양쪽에 있어 모델이 **"외워서"** studio 92점을 받음. 그런데 실제 사진은 35점 → **외운 거지 실력이 아님**(도메인갭). |

> **요점**: studio 0.92 vs wild 0.35 = 모델이 학습 사진을 외웠을 뿐 실제로는 일반화 못 함. **그래서 "wild 인식률"만 진짜 지표다.**

## 3. 스토리 — 6막

### 1막. 분류체계 다듬기 (exp06~10)
424개 AIHub 코드를 쓸 만한 클래스로 재설계(63→62→59). 헷갈리거나 라벨이 엉망인 클래스(chicken-galbi=가짜 닭갈비, 마라·탕수육·제육=데이터 빈약)를 정리. → **studio mAP 0.824 → 0.882로 상승.** 이때까진 "클래스 설계가 점수를 올린다"고 봄.

### 2막. 과적합 발견 (exp11)
사용자 지적으로 확인: 모델이 **train 0.99 vs val 0.88**로 외우고 있었음(과적합). 원인 = **데이터 기아**(cap 500이 8만 장을 버림). cap을 1500으로 올리니 과적합 완화(gap 0.117→0.097). **단 데이터가 풍부한 클래스만 개선(+0.023), 부족한 클래스는 정체(+0.0007).** → "부족클래스는 양 늘리기로 안 된다"의 첫 신호.

### 3막. 실데이터의 힘 (exp12~13)
selectstar(다른 출처) 실데이터를 takoyaki에 주입 → **인식률 9.5%→96%**. 11개 클래스로 확장해도 held-out 0.20→0.96. **실데이터가 약점클래스를 극적으로 살린다**는 증거. 그런데 여기서 **진짜 시험(wild)** 을 처음 도입 → 충격: **studio 0.89인데 wild 0.35.** 그리고 **탐지(박스)는 0.94로 멀쩡, 인식(분류)만 0.35** → 문제는 "어디 있나"가 아니라 "무엇인가"(분류).

### 4막. selectstar의 한계 (exp14)
selectstar로 클래스를 균등하게 채워봄. 결과 wild **0.317로 오히려 하락**. 보강한 클래스(0.57)는 올랐지만 **안 보강한 클래스가 잠식당해 악화(0.25)**. 원인 = selectstar도 **studio 도메인**이라 실환경 갭을 못 닫고, 22~25개 클래스만 커버해 나머지를 망침. **결론: studio 데이터로는 wild를 못 푼다. 진짜 실환경(wild) 데이터가 필요.** + 이때 **train/val 누수**도 발견(studio mAP는 가짜 점수).

### 5막. 분류체계 정리의 한계 (exp15~16a)
실데이터를 모으는 동안, 헷갈리는/모호한 클래스를 정리(taxo59→55→50: 냉라멘·전골·맑은국 등 drop, 빨간국·소스돈가스 등 merge). 혼동군 분석으로 후보를 골랐음. **하지만 wild는 안 움직였다**(exp16a−exp11 = −0.008, 통계적 노이즈). → **분류 설계로 짜낼 건 다 짜냈다** 확정.

### 6막. 진짜 실데이터의 입증 (exp16b) 🎯
실제 폰 사진(team·web 수집) **1,177장**을 학습에 섞음(전체의 단 2%). 결과:
- **wild 0.369 → 0.479 (+0.109)**, 부트스트랩 95% CI [+0.080~+0.139] = **0을 한참 벗어나 통계적으로 유의**(운 아님).
- realworld val 0.401 → 0.627 (단 학습과 같은 출처라 과대평가, wild가 정직한 지표).

→ **도메인갭은 실데이터로 풀린다.** 단 2%로 +30% 상대 개선이면, **더 모으면 더 오른다.**

---

## 4. 핵심 교훈 4가지

1. **studio mAP는 믿지 마라.** 누수로 부풀려진 가짜(0.92). **wild 인식률(0.48)만 진짜.**
2. **분류체계 설계는 한계가 있다.** 클래스 drop/merge는 wild를 못 올린다(노이즈). 정리·해석 가치만.
3. **약점은 데이터 양(cap·복제)으로 안 풀린다.** 같은 studio를 늘려봐야 과적합만. **다른 도메인(실환경) 데이터라야 함.**
4. **실데이터가 유일한 lever.** 2%만 섞어도 통계적으로 유의하게 wild가 오른다. 개선은 **클래스별 실데이터 양에 비례**(jjigae-red 0.31→0.62 vs seafood 정체).

### 부록: 통계 검증 방법(exp16)
작은 테스트셋(739장)에서 "한 번 채점이 운 아니냐"는 우려에 대해 **부트스트랩**(테스트셋 재표본 2,000회)으로 신뢰구간을 계산. 실데이터 효과는 CI가 0 밖(유의), 분류체계 효과는 CI가 0 포함(노이즈) → **재학습(멀티시드) 없이도 "무엇을 믿을지" 구분 가능**. 산출: `_eval_taxo50.py`, `_eval_taxo50_wild.csv`.
