# 음식 탐지 모델 — 성능 개선 여정 (멘토 피드백용)

**모델**: YOLO 계열 · 한식 음식 탐지 | **데이터**: AI Hub(단일디시), train≤500/val≤100 | **지표**: mAP50
**작성일**: 2026-06-02

---

## 1. 배경 (한 줄)
한식 음식 탐지 모델. **표준 튜닝으로 성능 천장을 못 뚫자, "데이터·택소노미·라벨 품질"로 방향을 틀어** 끌어올린 여정.

---

## 2. Phase 0 — 표준 튜닝 (exp01~05, 원본 50클래스): **변화 미미**
계획대로 **모델 크기 → 이미지 크기 → 증강 → 불균형 처리**를 순차 실험.

| 실험 | 처리 | mAP50 |
|---|---|---|
| exp01 | yolov8n baseline(full) | 0.846 |
| exp03 | yolo11s balanced 샘플링 | 0.823 |
| exp03 | yolov8n balanced 샘플링 | 0.790 |
| exp04 | yolov8n **증강**(minority aug) | 0.806 |
| exp05 | yolov8n **중복**(minority dup) | 0.804 |

→ **증강 0.806 ≈ 중복 0.804, balanced도 baseline 대비 개선 없음.** 모델·증강·샘플링 같은 표준 레버는 **0.79~0.85에서 횡보**, 유의미한 도약 없음.
**깨달음**: 천장의 원인은 하이퍼파라미터가 아니라 **클래스 설계·라벨 품질**. → 여기를 공략.

---

## 3. Phase 1 — 진짜 성능을 올린 3레버 (exp06+, taxo63 재설계)

**① 택소노미 재설계 (50 → 63클래스)**
혼동 덩어리를 의미단위로 분할·오염 정리 → **탕 0.56→0.84, 면 0.55→0.92** → baseline **0.824**

**② 모델 업그레이드 (yolo11s → yolo26s, 동일 batch16)**
exp08(yolo11s) 0.830 → exp07(yolo26s) **0.849** = 모델만으로 **+0.019**

**③ 라벨노이즈 정리 (chicken-galbi)**
500장인데 AP 0.53 → 데이터 부족 아닌 라벨 문제 의심 → 감사로 3개 이질코드 병합(후라이드·양념·치즈닭갈비, **진짜 닭갈비 0장**) 확인·정리 → **fried-chicken 0.670 → 0.896 (+0.226)** ✅

| 단계 | 실험 | 모델 / 택소노미 | mAP50 | mAP50-95 | 비고 |
|---|---|---|---|---|---|
| Phase0 | exp01 | yolov8n / **50클래스** | 0.846 | 0.819 | baseline(full) |
| Phase0 | exp02 | yolo11s / 50클래스 | (0.697) | (0.608) | 모델크기, 미완(ep3 탐색) |

## 5. 감사 방법 & 방법론 정직성
- **진단법**: 약한 클래스를 **소스코드별 분해 → GT박스 montage → 색·코드 분포 정량화**로 이질 병합 탐지
- **⚠️ 자기정정**: 혼동행렬 1차 해석에서 `category_id` 0-기반 오독(**off-by-one**) → 결론 폐기·재분석 (per-class AP 수치엔 영향 없음)

**② 다음 실험 방향**
