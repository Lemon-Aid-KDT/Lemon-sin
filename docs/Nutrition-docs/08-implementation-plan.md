### 1.2 Phase별 핵심 메시지

| Phase | 한 줄 메시지 | 핵심 산출물 |
|-------|------------|-----------|
| **0. 기획·조사** | "데이터·API·환경을 미리 확보한다" | 환경 세팅 완료 + 데이터 수급 계획 |
| **1. 핵심 산출식 PoC** | "회사 가이드를 그대로 코드로 만든다" | v1~v4 + 7-step Python 구현 + 단위 테스트 |

## 2. 역할(Role) 정의

> ⚠️ **중요**: 한 명이 여러 역할을 겸할 수 있고, 한 역할을 여러 명이 분담할 수도 있습니다. 팀 회의에서 강점·관심사에 따라 매핑하세요.

### 2.1 5가지 핵심 역할

#### 🔧 Backend Engineer (BE)
**핵심 책임**: 서버 사이드 로직 전체

| 영역 | 작업 |
|------|------|
| 알고리즘 구현 | v1~v4, BMR, TDEE, 7-step (07번 문서 기반) |
| API 설계 | FastAPI 라우터, Pydantic 스키마 |
| DB 설계 | PostgreSQL + TimescaleDB 스키마, Alembic 마이그레이션 |
| 캐싱 | Redis 통합 |
| 인증 | OAuth/OIDC Bearer JWT 검증, 스코프 권한 관리 |
| 단위 테스트 | pytest 50+ |

#### 📱 Mobile Engineer (ME)
**핵심 책임**: 사용자 접점 전체

| 영역 | 작업 |
|------|------|
| Flutter UI/UX | 5종 출력 화면, 입력 폼, 권고 카드 |
| 헬스 데이터 | HealthKit / Health Connect 연동 (`health` 패키지) |
| 카메라 | 영양제 사진 촬영, 갤러리 선택, 크롭 |
| 상태 관리 | Riverpod / Provider |
| API 통신 | Dio + Retrofit, 에러 처리 |
| 라우팅 | go_router |
| 앱 빌드 | iOS / Android 양쪽 빌드, 스토어 메타데이터 |

#### 🤖 AI/ML Engineer (AI)
**핵심 책임**: 갭 영역 알고리즘 + 로컬 LLM/OCR 파이프라인 통합

| 영역 | 작업 |
|------|------|
| OCR 파이프라인 | Google Cloud Vision 통합, 폴백 로직 |
| LLM 프롬프트 | Ollama 로컬 LLM + JSON Schema, 영양제·식단 파싱 |
| 부족 영양소 진단 | 알고리즘 ⓒ (07번) |
| 목적별 분석 | 알고리즘 ⓓ (눈/간/피로 매트릭스) |
| 식단 인식 | Phase 3 — 텍스트 → 이미지 |
| 정확도 평가 | 100장 테스트셋, 메트릭 측정 |

#### 📊 Data / Domain (DD)
**핵심 책임**: 데이터·도메인 지식 통합

| 영역 | 작업 |
|------|------|
| KDRIs 디지털화 | PDF → CSV/JSON 변환 (30종 영양소 × 연령·성별) |
| 식약처 API 통합 | 식품영양성분 API + 건강기능식품 원료 DB |
| AI Hub 데이터셋 | 음식 이미지 데이터셋 신청·다운로드 |
| 의료 자문 코디 | 의료자문위 검토 일정·자료 정리 |
| 영양·운동 도메인 | 알고리즘 검증, 가이드라인 인용 |
| 표현 가이드 | 의료법·약사법 표현 검수 |

#### 📋 Project Manager / QA (PM)
**핵심 책임**: 일정 관리 + 품질 보증 + 발주처 커뮤니케이션

| 영역 | 작업 |
|------|------|
| 일정 관리 | 주간 마일스톤 점검, 지연 시 우선순위 재조정 |
| Issue / PR 관리 | GitHub 라벨링, 리뷰 분배 |
| 통합 테스트 | E2E 시나리오, QA 체크리스트 |
| 문서 동기화 | docs/ 갱신, README 유지 |
| 발표 자료 | 슬라이드, 시연 시나리오 |
| 발주처 커뮤니케이션 | 레몬헬스케어 미팅 |

### 2.2 역할 매핑 가이드

| 팀원 수 | 권장 매핑 |
|--------|---------|
| **2명** | A: BE+AI / B: ME+DD+PM (모두 분담) |
| **3명** | A: BE / B: ME+DD / C: AI+PM |
| **4명** | A: BE / B: ME / C: AI+DD / D: PM |
| **5명** | 1:1 매핑 (가장 이상적) |
| **6명+** | 1:1 매핑 + BE/ME 영역에 1명 추가 |

> 💡 **핵심 원칙**: 각 역할에 **최소 1명의 책임자(Owner)** 가 있어야 함. 겸직은 가능하지만 책임자가 모호하면 작업 누락 발생.

*"회사 가이드(v1~v4, 7-step)를 그대로 코드로 만들어 가이드 PPTX의 모든 계산 예시를 단위 테스트로 검증한다"*

#### 주차별 마일스톤

| 주차 | 마일스톤 | 검증 |
|------|---------|------|
| **W2** | BMI 분류 + v1 (권장걸음수, 기본점수) 구현·테스트 | "50대 여성 비만1 = 7,524보, 7,000보 → 77.5점" 검증 통과 |

| **W4** | 7-step 체중예측 + KDRIs 룩업 + API 엔드포인트 | "30일 후 67.19kg, 60일 후 79.39kg" (가이드 예시2) 검증 통과 |

| PM | 주간 점검, 이슈 트리아지 | 〃 | E2E 시연 시나리오 초안 |

## 7. 발표·시연 준비

### 7.1 시연 시나리오 (페르소나 B "김건강" 사용)

```
[화면 1] 앱 시작 — 만성질환 정보 입력
  "고혈압, 당뇨 전단계" 선택

[화면 2] 영양제 4종 사진 촬영
  종합비타민 / 오메가3 / 칼슘 / 마그네슘

[화면 3] 자동 분석 결과
  "비타민C 1000mg, 비타민D3 25μg, ..." 자동 추출

[화면 4] 5종 출력 대시보드
  ① 부족 영양소: "철분, 비타민B12 부족"
  ② 권장 섭취량: 만성질환자 맞춤 KDRIs
  ③ 체중 변화 예측: "1주 -0.3kg, 1개월 -1.2kg, 3개월 -3.5kg"
  ④ 운동 권고: "권장 7,524보 / 활동점수 v4 = 87.2"
  ⑤ 목적별 분석: "눈건강 점수 65/100"

[화면 5] 면책 고지 + 의료진 상담 권고
  "이 정보는 진단·처방을 대체하지 않습니다"
```

### 7.2 발표 핵심 메시지 3가지

1. **차별화 명확** — *"필라이즈가 못하는 만성질환자 + 의료데이터 영역"*
2. **검증 가능한 알고리즘** — *"회사 가이드 PPTX의 모든 계산 예시가 단위 테스트로 통과"*
3. **즉시 가동 가능한 시스템** — *"GitHub Actions CI, Branch Protection, 의료자문위 검토까지 완비"*

### 7.3 Q&A 대비

예상 질문 + 답변 키워드:
