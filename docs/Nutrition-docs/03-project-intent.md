## 1. 기획 출발점 — 시장의 빈자리

### 1.1 기존 서비스 지도

```
[일반 영양제 분석]            [식단 분석]              [처방약 관리]
   필라이즈 75만 MAU           CalZen 300만+            마이약
   홈스픽                       식사진 (한국 음식)
       │                           │                       │
       └────────┬──────────────────┴───────────────────────┘
                │
        모두 "사용자 입력에 의존" + "단일 영역 특화"
                │
                ▼
        🎯 빈자리: 만성질환자 + 의료 데이터 + 통합 분석
```

### 1.2 시장의 빈자리

| 빈자리 | 이유 |
|--------|------|
| **만성질환자 중심 통합 분석** | 기존 서비스는 일반 사용자용 — 만성질환자는 영양제·운동·식단 권고가 더 까다로움 |
| **의료 데이터 연계 가능성** | 진료기록·처방·검진 데이터까지 통합할 수 있는 곳은 LDB 보유 회사뿐 |
| **임상 근거 기반 가중치** | v4 만성질환 가중치 같은 정교한 알고리즘은 회사 가이드에만 정의됨 |
| **신뢰 가능한 발주처** | 상급종합병원 80% 점유 + 코스닥 상장 — 의료적 신뢰도 ↑ |

> 💡 **핵심 인사이트**: 필라이즈를 "정면 추격"하는 것이 아니라, **"필라이즈가 갈 수 없는 곳"** 으로 가는 전략이다.

## 2. 차별화 전략 — 5가지 무기

### 무기 ①. 의료기관 네트워크 (LDB) — **가장 강력한 차별화**

| 항목 | 본 프로젝트 | 필라이즈 |
|------|-----------|---------|
| 사용자 진료 기록 연계 | ✅ 가능 (LDB) | ❌ 불가 |
| 처방 정보 자동 반영 | ✅ 가능 (마이데이터) | ❌ 불가 |
| 의료기관 협력 | ✅ 130여 곳 | ❌ 없음 |

> 📌 **이것 하나만으로도 만성질환자에게는 차별화가 명확**해진다. "내 영양제가 약과 충돌하는지", "처방 결과가 어떻게 영양 상태를 바꿨는지"는 진료 데이터 없이는 답할 수 없다.

### 무기 ②. 만성질환 v4 가중 알고리즘 — **임상적 정교함**

회사 가이드에 정의된 v4 만성질환 가중치:
- 당뇨 +0.10 / 고혈압 +0.10 / 심혈관 +0.15 / 관절 +0.15 / 호흡기 +0.10 (최대 ×1.3)
- "동일 활동 대비 만성질환자에게 더 높은 건강 효과 부여"

→ 이 알고리즘은 일반 헬스케어 앱에는 없다. **만성질환자에게 *"당신의 7,000보는 일반인의 9,000보와 같은 가치"* 라고 말할 수 있는 유일한 앱**.

### 무기 ③. 770만 검증된 사용자 베이스

- 청구의신을 통해 이미 의료 행동 데이터 축적
- 별도의 사용자 확보 비용(CAC) 없이 즉시 마케팅 가능
- 신규 진입자(필라이즈 등)가 따라올 수 없는 출발점

### 무기 ④. 공식 표준 데이터 활용

- 한국영양학회 KDRIs 2020 기반 30종 영양소
- 보건복지부·식약처 데이터
- 한국·아시아 BMI 분류 (서양 25 미만 정상과 다른 23~ 과체중 분류)
- → "한국인 맞춤" 이라는 명확한 포지셔닝

### 무기 ⑤. 5종 출력 통합

```
┌──────────────────────────────────────────────┐
│  사진 한 장 → 5가지 답을 한 번에            │
│                                                │
│  ① 부족 영양소     ② 권장 섭취량              │
│  ③ 체중 변화 예측   ④ 운동 권고                │
│  ⑤ 목적별 분석 (눈·간·피로)                   │
└──────────────────────────────────────────────┘
```

기존 앱들은 1~2개 영역에 특화되어 있다. **하나의 앱에서 5종을 모두 통합**하는 것이 차별화 포인트.

## 3. 타겟 페르소나

### 3.1 1차 핵심 페르소나 — B형: "**김건강** (52세, 만성질환 관리자)"

```
┌─────────────────────────────────────────────────────────────┐
│  👤 김건강 / 52세 / 남성 / 부산 거주                          │
│  💼 중소기업 부장 (사무직 25년차)                            │
│  👨‍👩‍👧 가족: 아내, 자녀 2명 (대학생, 고등학생)                │
│  📱 디지털 친화도: 중간 (앱 설치는 신중하게 결정)             │
└─────────────────────────────────────────────────────────────┘
```

#### 건강 상황
- 2년 전 **고혈압 진단**, 작년 **당뇨 전단계** 진단
- 약물: 혈압약 1종 (매일 복용)
- 영양제: **4종 동시 복용** (종합비타민, 오메가3, 칼슘, 마그네슘)
- 체중: 178cm / 84kg → BMI 26.5 (비만 1단계)
- 활동: 평일 6,500보 / 주말 3,000보

#### 페인 포인트 (Pain Points)
| 분류 | 구체적 고민 |
|------|----------|
| **영양제 혼란** | "약사 추천 / 인터넷 추천 / 아내 추천이 다 다름. 4종이나 먹는데 진짜 맞는 건가?" |
| **약물 상호작용 우려** | "혈압약과 영양제가 충돌하면 어쩌지?" |
| **만성질환자 맞춤 정보 부족** | "운동 권고도 일반인 기준 → 나에게 맞는지 모름" |
| **합병증 공포** | "당뇨로 진행되면 어쩌지? 지금 뭘 해야 하나?" |
| **가족 안심** | "아내·자녀가 걱정함. 객관적 데이터로 안심시키고 싶음" |

#### Job-to-be-Done (실현하고 싶은 일)
1. 내가 먹는 영양제가 **약과 충돌하지 않는지** 확인하고 싶다
2. 내 건강 상태에 맞는 **만성질환자용 권장량**을 알고 싶다
3. 합병증 예방을 위해 **무엇을 더 하고 덜 해야 하는지** 알고 싶다
4. 가족에게 **객관적 수치로** 보여주고 싶다

#### 성공의 모습 (이 앱을 쓰면)
- 4종 영양제 사진 한 번 찍으면 즉시 분석 + 약과의 상호작용 검토
- "당뇨 진행 위험 ↓" 같은 정량적 피드백
- 의료진과 데이터 공유 가능 (LDB 연계 시)

---

### 3.2 2차 확장 페르소나 — A형: "**박직장** (38세, 예방 단계 직장인)"

```
┌─────────────────────────────────────────────────────────────┐
│  👤 박직장 / 38세 / 남성 / 서울 거주                          │
│  💼 IT 회사 팀장 (앉아서 일하는 시간 8시간+)                   │
│  👨‍👩‍👦 가족: 아내, 자녀 1명 (5세)                              │
│  📱 디지털 친화도: 높음 (앱 설치·구독 익숙)                   │
└─────────────────────────────────────────────────────────────┘
```

#### 건강 상황
- 정기 검진에서 **콜레스테롤 ↑, 공복혈당 경계** 통보
- 영양제: 종합비타민 1종 (간헐적)
- 체중: 175cm / 82kg → BMI 26.8 (비만 1단계 진입)
- 활동: 평일 5,000보 미만 / 주말 가끔 산책

#### 페인 포인트
- "회식 + 야근 + 운동 부족"의 악순환
- 영양제 정보 과잉 — 뭘 먹어야 할지 결정 못함
- 다이어트 시도 → 매번 실패
- **"시간이 없다"**

#### Job-to-be-Done
1. **최소 시간 투자**로 건강 관리하고 싶다
2. **만성질환으로 진행되기 전에** 예방하고 싶다
3. 효율적인 다이어트 방법을 찾고 싶다

#### 성공의 모습
- 1주/1개월/3개월 후 체중 예측으로 동기 부여
- 야근 후 늦은 밤에도 사진 한 장으로 식단 기록
- "이대로 가면 3년 후 당뇨" 같은 경고로 행동 변화

### 3.3 페르소나 B vs A — 우선순위 차이

| 항목 | B (1차 핵심) | A (2차 확장) |
|------|-------------|-------------|
| **핵심 기능** | 영양제 분석 + 약물 상호작용 + 만성질환 가중 | 체중 예측 + 식단 분석 + 운동 권고 |
| **UI 우선순위** | 명확성·안정감·큰 글씨 | 빠른 입력·시각화·동기부여 |
| **차별화 무기** | LDB 의료 데이터 연계 (무기 ①) | 5종 출력 통합 (무기 ⑤) |
| **마케팅 채널** | 의료기관·약국 제휴, 가족 추천 | 직장인 커뮤니티, 헬스 인플루언서 |

## 4. 핵심 가치 제안 (Value Proposition)

### 4.1 페르소나 B용 (1차)

> **"만성질환을 가진 분이라면, 일반 영양제 앱은 충분하지 않습니다.**
> **건강의신은 진료 데이터까지 연결해 약·영양제·식사·운동을 함께 봅니다."**

핵심 메시지:
- ✅ 약과의 상호작용 검토
- ✅ 만성질환자 맞춤 권장량 (KDRIs + v4 가중)
- ✅ 의료진과 데이터 공유 (향후)
- ✅ 합병증 예방 시뮬레이션

### 4.2 페르소나 A용 (2차)

> **"바쁜 직장인을 위한, 사진 한 장으로 끝나는 건강 관리.**
> **3개월 후 당신의 체중·건강 상태를 미리 보세요."**

핵심 메시지:
- ✅ 사진 한 장 → 5가지 분석 (영양제·식단·체중·운동·목적별)
- ✅ 1주/1개월/3개월 후 체중 예측
- ✅ 만성질환 진행 위험 사전 경고
- ✅ HealthKit·Health Connect 자동 연동 → 입력 부담 ↓

## 5. 성공 시나리오 (3년 로드맵)

### Year 1 (2026): PoC + 베타
- 학생 팀: Phase 0~4 완료, 발표
- 회사: 만성질환자 100명 클로즈드 베타 (의료기관 협력)
- 핵심 검증: RQ1~RQ4 답변 완료

### Year 2 (2027): 정식 출시 + 의료기관 연계 시범
- 건강의신 정식 출시
- 5~10개 의료기관과 LDB 연동 시범 (페르소나 B 진입점)
- KPI: MAU 50만, 만성질환자 비율 30%+
- ISMS-P 인증 추진

### Year 3 (2028): 마이데이터 통합 + DTx 진입 검토
- 의료 마이데이터 채널 연동 (보건복지부 사업 연계)
- DTx (디지털치료기기) 인허가 검토 시작
- 글로벌 확장 시범 (일본·동남아)

## 7. 전제 조건 (Critical Assumptions)

본 기획이 성립하기 위한 전제 조건들. 이 중 하나라도 무너지면 기획을 재검토해야 한다.

| # | 전제 | 검증 방법 |
|---|------|---------|
| A1 | 의료자문위 구성이 가능하다 | 회사가 130여 의료기관 네트워크 보유 → 가능성 매우 높음 |
| A2 | 사용자가 만성질환 정보를 자발적으로 입력한다 | 페르소나 B는 이미 약 복용 중 → 동기 충분, UI에서 인센티브 제공 |
| A3 | HealthKit/Health Connect가 신뢰할 만한 걸음수를 제공한다 | 이미 검증된 플랫폼 → 안정적 |
| A4 | 회사 가이드(v1~v4, 7-step)가 임상적으로 타당하다 | Mifflin-St Jeor·KDRIs 모두 표준 → 타당, 단 한국인 보정 필요 |
| A5 | OCR + LLM 조합으로 영양제 라벨 정확도 ≥ 85% 달성 가능 | 업계 표준 (Google Vision 92~98% 보고) → 가능성 높음 |
| A6 | 학생 팀의 10주 일정이 충분하다 | Phase 0~4 마일스톤 도달 가능성 — 진행 중 점검 필요 |
