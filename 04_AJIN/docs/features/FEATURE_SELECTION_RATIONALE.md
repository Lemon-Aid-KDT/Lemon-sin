## 1. 왜 이 6가지 기능인가

### 1.1 아진산업의 현실적 과제에서 출발

아진산업은 1976년 창립 이래 약 50년간 현대자동차그룹 1차 협력사로서 자동차 차체 부품을 공급해온 중견기업이다. 2025년 기준 매출 1조 886억 원, 임직원 649명, 국내 12개 사업장 + 해외 7개 법인을 운영하며, 전기차 전장부품(EWP, OBC, CCH)으로 사업을 확장하고 있다.

이 규모와 사업 다각화는 곧 **업무 복잡성의 기하급수적 증가**를 의미한다. 아진산업이 일상적으로 직면하는 3대 과제를 분석하여 기능을 선정하였다.

| 과제                               | 현장 실태                                                                                                                                  | AI 해법 (기능)                                              |
| ---------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------- |
| **문서 과부하**              | 완성차 품질 보고서, ECR/ECN, PPAP, 8D, 납기 이메일이 21개 계열사 x 4개국에 걸쳐 발생. 담당자가 양식을 찾고 작성하는 데만 하루 1~2시간 소요 | **기능 A** (인원 검색) + **기능 B** (문서 작성 + CSV/XLSX 다운로드) |
| **신입사원 온보딩 지연**     | 프레스/용접/도장/조립 공정 용어, IATF 16949, APQP, SPC 등 학습 부담. 신입이 실무 투입까지 평균 3~6개월 소요                                | **기능 C** (AI 업무 도우미 + 파일 업로드 분석)               |
| **법규 컴플라이언스 리스크** | 산안법, EU REACH, 관세 정책 등 지속 변화. 미준수 시 납품 자격 박탈, 과징금. 기존엔 외부 컨설팅(1,000만 원+) 의존                           | **기능 D** (법규 모니터링 + 업데이트 추적 + CSV 내보내기)    |
| **설비 고장 대응 지연**      | 프레스 7종, 용접기, 로봇 등 수십 대 설비. 에러코드 201건, 고장 시 경험자만 대응 가능. 신입/비전공자 대응 불가                              | **기능 F** (설비/공정 AI + SPC 데이터 관리)                  |
| **인사 관리 비효율**         | 649명 x 30개 부서 x 6단계 직급. 계정 생성/권한 부여/보안 감사 수동 관리                                                                    | **기능 E** (인사 관리 + 보안 감사 통합 + 이력 다운로드)      |

### 1.2 SILLI 경진대회 주제와의 직접 매칭

SILLI 경진대회는 "제조 현장 및 일상 업무에 인공지능을 접목한 통합 업무 효율화 플랫폼"을 평가한다. 6대 기능은 경진대회의 5개 평가 축에 **빠짐없이 대응**하도록 설계되었다.

| 경진대회 평가 축      | 대응 기능  | 구체적 기여                                                               |
| --------------------- | ---------- | ------------------------------------------------------------------------- |
| **업무 효율화** | 기능 A + B | 인원 검색 0.3초(FTS5+시맨틱), 문서 초안 Few-shot RAG 자동 생성, **CSV/XLSX 즉시 다운로드** (v3.5) |
| **안전성 향상** | 기능 D + F | 산안법 변경 실시간 감지 + 프레스 안전거리 시뮬레이션 + Nelson 8 Rules SPC + **법규 업데이트 추적 4탭** (v3.5) |
| **품질 개선**   | 기능 B + F | PPAP/8D 문서 품질 자동 평가(5기준 100점) + SPC Cpk 예측 + 에러 연쇄 분석 + **SPC 데이터 생성기** (v3.5) |
| **생산성 향상** | 기능 C     | 신입 온보딩 3~6개월 → 수주 내 조기 전력화 (SOP 8종 + 퀴즈 + 협업 가이드 + **파일 업로드 분석**) |
| **투자비 절감** | 기능 D + E | 외부 법률 컨설팅 대체 + 인사 관리 자동화 + 보안 감사 AI + **로그인 이력 CSV 감사 보고서** (v3.5) |

### 1.3 왜 범용 AI가 아닌 맞춤형 6대 기능인가

ChatGPT나 Claude 같은 범용 AI는 아진산업의 실무에 직접 적용하기 어렵다.

| 한계                            | 설명                                                       | 본 프로젝트의 해법                                                  |
| ------------------------------- | ---------------------------------------------------------- | ------------------------------------------------------------------- |
| **사내 맥락 부재**        | 아진산업 고유의 문서 양식, 업무 프로세스, 조직 구조를 모름 | 31개 부서 프로필 + 297개 용어집 + 11종 문서 양식 내장               |
| **도메인 지식 부족**      | 자동차 부품 제조업 품질 규격(IATF 16949) 이해 부족         | SOP 8종 + 에러코드 201건 + SPC Nelson 8 Rules + 금형 수명 예측      |
| **법규 실시간 반영 불가** | 학습 데이터 시점 한계                                      | 9종 크롤러 + 변경 감지 + 리스크 스코어링 + **법규 업데이트 추적 탭** (v3.5) |
| **보안 문제**             | 사내 기밀을 외부 클라우드에 전송                           | **온프레미스 완전 구축** (Ollama — Qwen 3.5/EXAONE/Gemma 4/GPT-OSS, 외부 API 미사용) |

## 2. 기능별 상세 선정 근거

### 2.1 기능 A: 인원 검색 및 조직도 — "누구에게 물어봐야 하지?"

**선정 이유:** 649명 x 30개 부서 조직에서 "이 업무는 누구한테 물어봐야 하나?"는 **가장 빈번한 질문**이다. 특히 신입사원은 담당자를 찾는 것 자체가 업무의 병목이 된다.

**아진산업 맥락:**

- 12개 국내 사업장에 분산된 인력 → 같은 팀이라도 경산 본사와 경주 공장에 분리
- 부서 약어/별칭 혼용 (품질관리팀/품질보증팀, 연구소/기술연구소)
- 완성차 SQ 담당자, ECN 담당자 등 역할 기반 검색 필요

**핵심 기술:** FTS5 전문 검색 + ChromaDB 시맨틱 하이브리드(RRF) + ML 의도분류(TF-IDF+LR)

**전 직원 접근:** 인원 검색은 직급/부서 무관하게 **모든 임직원이 필요**로 하는 기초 기능이므로 접근 제한 없음.

---

### 2.2 기능 B: 문서 검색 및 작성 — "이 양식 어디 있지? 어떻게 쓰지?"

**선정 이유:** 아진산업의 핵심 업무 중 상당 부분이 **문서 작성**이다. 8D Report, ECN, PPAP, 이메일, 회의록 등 정형화된 문서가 매일 발생하지만, 양식을 찾고 올바르게 작성하는 데 시간이 소요된다.

**아진산업 맥락:**

- 현대/기아 SQ에 제출하는 품질 문서는 형식 오류만으로도 반려됨
- 21개 계열사 간 이메일/보고서 커뮤니케이션 양이 방대
- 신입사원이 문서 양식과 작성 관례를 익히는 데 수개월 소요

**핵심 기술:** Few-shot RAG (584건 인덱싱, 아진 스타일 자동 생성) + 문서 품질 자동 평가(5기준 100점) + CC 자동 추천 + **CSV/XLSX 다운로드** (v3.5: 마크다운 테이블 자동 파싱 → 구조화 데이터 내보내기)

**전 직원 접근:** 문서 작성은 사무직 전원의 일상 업무이므로 접근 제한 없음.

---

### 2.3 기능 C: AI 업무 도우미 — "이게 뭐예요? 어떻게 해요?"

**선정 이유:** SILLI 경진대회의 핵심 취지인 **"생산성 향상"**에 가장 직접적으로 기여하는 기능이다. 제조업 전반의 높은 신입사원 조기 퇴사율 문제와 직결된다.

**아진산업 맥락:**

- 프레스/용접/도장/조립 공정별 전문 용어 수백 개
- IATF 16949, APQP, SPC, FMEA, Control Plan 등 품질 규격 학습 부담
- "프레스 트라이가 뭔가요?", "8D 올려달라는데 뭘 준비하지?" 같은 질문이 매일 발생
- 선배에게 반복 질문하면 업무 방해 → AI로 대체하면 양쪽 모두 시간 절약

**핵심 기술:**

- SOP 8종 (금형교체, 용접검사, CNC가공, PPAP, 8D, ECN, 프레스 트라이, 금형 입고)
- 부서 간 협업 시나리오 5종 (8D/ECN/SPC/PPAP/안전점검)
- 듀얼 모드 (교육 모드: 용어 학습 / 업무 모드: 액션 실행)
- 퀴즈 자동 생성 + 오답 재학습 경로
- **다운로드 영구화** (v3.5): DOCX/XLSX/CSV/TXT 다운로드 버튼이 rerun 후에도 유지 (session_state 기반)
- **파일 업로드 분석**: CSV, XLSX, PDF, DOCX, 이미지 등 20+ 확장자 지원 + Gemma 4 비전 모델 이미지 분석

**전 직원 접근:** 신입뿐 아니라 부서 이동, 직무 전환, 타부서 협업 시에도 필요하므로 접근 제한 없음. 단, **소속 부서 기반 맞춤 응답** (부서 selectbox 고정)으로 관련성 보장.

---

### 2.4 기능 D: 법규/규정 모니터링 — "이 규제 우리 공장에 해당되나?"

**선정 이유:** 아진산업은 1차 협력사로서 법규 미준수가 곧 **납품 자격 박탈**이나 **과징금**으로 직결된다. 기존에는 외부 법률 컨설팅(1,000만 원+, 2~5개월)에 의존했으나, AI로 실시간 모니터링이 가능하다.

**아진산업 맥락:**

- 산업안전보건법: 12개 국내 사업장 전체 적용 (프레스 안전거리 등)
- EU REACH: 유럽 수출 부품의 화학물질 규제 (도금 공정 크롬산 등)
- 미국 관세: JOON INC(조지아) → HMGMA 공급 원가에 직접 영향
- ESG: 탄소중립, 지속가능경영 보고 의무 확대

**부서별 접근 제한 이유 및 선정 기준:**

| 허용 부서 (8개)      | 선정 근거                                                          |
| -------------------- | ------------------------------------------------------------------ |
| **ESG경영팀**  | ESG 전략, 탄소중립, 지속가능경영 — 규제 대응의**총괄 부서** |
| **품질보증팀** | IATF 16949, PPAP, 8D —**완성차 SQ 직접 대응** 부서          |
| **품질경영팀** | ISO/IATF 인증 관리 — 품질경영 시스템**인증 유지** 담당      |
| **안전보건팀** | 산안법, MSDS, 작업환경측정 —**안전 법규 직접 관리** 부서    |
| **구매팀**     | 원자재/부품 구매 — 소재 규제(REACH, RoHS)**공급망 영향**    |
| **해외지원팀** | 해외법인 지원 —**글로벌 규제** (미국 관세, EU 규정) 대응    |
| **상생협력팀** | 협력사 동반 성장 — 협력사에 대한**규제 전파** 역할          |
| **영업팀**     | OEM 영업 — 고객(현대/기아)의**규제 요구사항 접수** 창구     |

**비허용 부서 근거:** IT전략팀, 재무팀, 회계팀 등은 법규 규제의 직접 대상이 아니며, 규제 정보가 필요한 경우 해당 부서를 통해 간접 확인. 불필요한 정보 노출을 방지하여 **업무 집중도**를 유지한다.

**v3.5 구조 개선:**
- **4탭 구조**: 법규 모니터(6서브탭) / **법규 업데이트**(메인 탭 승격) / 사업장 / 법규 문서
- **법규 업데이트 탭 통합**: 시나리오 TOP-3 대시보드 + 변경 감지 메트릭 + 필터 + CSV 내보내기 + 전후 비교
- **크롤링 UI 개선**: 실행 버튼 상단 이동 + 빠른 현황 요약 라인
- **한글 인코딩 수정**: `_safe_truncate()` 안전 잘림 + UTF-8 PRAGMA + U+FFFD 19개 복원

### 2.5 기능 E: 인사 관리 — "계정 만들어주세요, 권한 변경해주세요"

**선정 이유:** 649명 x 30개 부서 x 6단계 직급 체계를 안전하게 관리하려면 체계적인 인사 관리 시스템이 필수이다. 특히 AI 시스템 자체의 **접근 제어와 보안 감사**를 위해 반드시 필요하다.

**아진산업 맥락:**

- 33명 테스트 계정으로 6단계 RBAC(역할 기반 접근 제어) 운영
- JWT 인증 + bcrypt 암호화 + 5회 실패 잠금
- 비밀번호 정책 강화 (8자+대소문자+숫자+특수문자)
- 보안 감사: 무차별 대입 감지, 야간 접근 감지, 비활성 계정 감지

**Tier 기반 접근 제한:**

| Tier            | 역할                 | 접근 범위                | 탭 구성 (v3.5)           | 근거                  |
| --------------- | -------------------- | ------------------------ | ------------------------ | --------------------- |
| Tier 1 (SELF)   | 전 직원              | 본인 정보/비밀번호 변경  | 보안(권한 있을 시)        | 기본 자기 관리        |
| Tier 2 (TEAM)   | 팀장급(차장+)        | 팀원 목록 조회           | 내 팀 + 보안(권한 시)     | 팀 관리 필요          |
| Tier 3 (DEPT)   | 과장급 + 총무인사팀  | 사용자 연락처/부서 수정  | 사용자/보안/분석/인사통계 | HR 업무 위임          |
| Tier 4 (SYSTEM) | HR_ADMIN / SYS_ADMIN | 계정 생성/삭제/역할 변경 | 사용자/생성/보안/분석/인사통계/도구 | 시스템 보안 최고 권한 |

**v3.5 개선:**
- **이력 탭 → 보안 탭 통합**: Tier 4(7→6탭), Tier 3(5→4탭) — 보안 탭 하단에 접이식 상세 이력 배치
- **로그인 이력 CSV/XLSX 다운로드**: 날짜 필터 + 타임스탬프/사번/부서/IP/User-Agent 전체 컬럼

### 2.6 기능 F: 설비/공정 AI — "이 에러코드 뭔가요? 다음에 뭐가 고장나나요?"

**선정 이유:** 아진산업의 **핵심 경쟁력은 제조 현장**이다. 프레스, 용접기, 로봇, 사출기, CNC, 레이저 등 수십 대 설비의 안정적 가동이 납기와 품질을 좌우한다. 설비 고장 시 경험 많은 기술자만 대응할 수 있는 현실을 AI로 해소한다.

**아진산업 맥락:**

- 에러코드 201건 (7종 설비 유형)
- 금형 25건 관리 (수명 추적, 교체 예측)
- SPC 5공정 실시간 모니터링 (EWP, CCH, OBC 등 전기차 핵심 부품)
- 프레스 라인 안전거리 관리 (산안법 적용 대상)

**부서별 접근 제한 이유 및 선정 기준:**

| 허용 부서 (14개)       | 선정 근거                                                        |
| ---------------------- | ---------------------------------------------------------------- |

- **데이터 생성기**: 5개 공정 일괄 재생성 (샘플 수/시드 설정, 트렌드/평균이동/이상치 자동 주입)

## 3. 접근 권한 설계 철학

### 3.1 3원칙

| 원칙                               | 설명                                                                                                        | 적용 예                                                          |
| ---------------------------------- | ----------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------- |
| **전원 접근 기능** (A, B, C) | 인원 검색, 문서 작성, AI 업무 도우미는**직급/부서 무관하게 모든 임직원**이 필요로 하는 기초 업무 도구 | 신입 사원도 첫날부터 사용 가능                                   |
| **부서 기반 제한** (D, F)    | 법규/설비 정보는**해당 업무와 직접 관련된 부서만** 접근하여 정보 과부하 방지 + 책임 소재 명확화       | 설비 AI는 생산/기술 14개 부서만, 법규는 품질/안전/ESG 8개 부서만 |
| **직급 기반 계층** (E)       | 인사 정보는**민감도에 따라 4단계로 분리**하여 최소 권한 원칙(Principle of Least Privilege) 적용       | 본인 정보 < 팀원 조회 < 부서 수정 < 계정 생성                    |

### 3.2 설계 기준: 왜 이 부서가 포함/제외되었는가

**포함 기준:**

1. 해당 기능의 데이터를 **직접 생성하거나 소비**하는 부서
2. 해당 영역의 **의사결정 권한**을 가진 부서
3. **법적 책임** (산안법 안전보건팀, REACH ESG경영팀 등)이 있는 부서

**제외 기준:**

1. 해당 데이터와 **업무 접점이 없는** 부서 (재무팀이 에러코드를 볼 필요 없음)
2. **간접 확인으로 충분**한 부서 (영업팀은 규제 정보를 ESG/품질 통해 확인)
3. **정보 과부하 방지** — 불필요한 메뉴를 숨겨 해당 부서의 핵심 업무에 집중하도록 지원

## 4. 결론: 아진산업을 위한 맞춤형 AI

본 프로젝트의 6대 기능은 아진산업이 **실제로 직면하는 5대 과제**(문서 과부하, 온보딩 지연, 법규 리스크, 설비 고장, 인사 관리)에 **1:1 대응**하도록 설계되었다. 범용 AI가 아닌 아진산업의 조직 구조(8본부 30개 부서), 업무 프로세스(PPAP/8D/ECN/SPC), 설비 현황(7종 201 에러코드)을 **내장한 맞춤형 온프레미스 AI**이다.

v3.5에서는 현장 피드백을 반영하여 **데이터 내보내기**(CSV/XLSX 전체 적용), **다운로드 안정성**(AI 도우미 rerun 생존), **법규 추적 가시성**(4탭 승격), **보안 감사 효율화**(탭 통합+이력 다운로드), **SPC 데이터 자율 관리**(생성기+업로드), **LLM 모델 최적화**(5개 패밀리 11개 모델, 68GB 절약)를 달성하였다.

SILLI 경진대회가 추구하는 "제조 현장에 AI를 접목한 실질적 업무 효율화"의 취지에 부합하며, **신입사원이 첫날부터 사용할 수 있는 기능**(A, B, C)과 **전문 부서가 깊이 있게 활용하는 기능**(D, E, F)의 이중 구조로 **전 임직원 649명의 업무 생산성 향상**에 기여한다.
