# 아진산업 AI 어시스턴트 — 실 데이터 요청서

> **프로젝트**: AJIN AI ASSISTANT v2.5
> **작성일**: 2026-03-26
> **목적**: 시뮬레이션 데이터 → 실 데이터 전환을 위한 아진산업 제공 데이터 목록

---

## 개요

현재 AI 어시스턴트는 **100% 시뮬레이션(가상) 데이터**로 구동되고 있습니다.
실제 업무 환경에서 정확하게 동작하려면 아진산업의 실 데이터가 필요합니다.

아래 목록은 **우선순위별**로 정리되어 있으며, **P0(최소 필수)** 데이터만 확보해도 핵심 기능이 실데이터 기반으로 전환됩니다.

> **개인정보 보호**: 모든 데이터는 담당자 실명을 마스킹 처리하여 제공 가능합니다.

## 🔴 P0 — 최소 필수 (핵심 기능 동작에 필수)

## 데이터 제공 방법

### 권장 형식

| 데이터 유형 | 권장 형식 | 비고 |
|-----------|----------|------|
| 문서 양식/샘플 | DOCX, PDF, HWP | 원본 양식 그대로 |
| 목록/표 데이터 | Excel (.xlsx), CSV | 사원 명부, 인증 목록 등 |
| 절차서/매뉴얼 | PDF, DOCX, MD | SOP, 교육 자료 |
| 이미지/도면 | PNG, PDF | 조직도, 공정도 |
| 용어집 | Excel, JSON | 약어/용어 목록 |

### 개인정보 보호 가이드

- **마스킹 필요 항목**: 사원 실명, 연락처, 이메일 주소, 주민등록번호
- **마스킹 불필요**: 부서명, 직급명, 부품명, 공정명, 규격번호
- **대체 방법**: 실명 대신 "홍길동", "김OO" 등 가명 사용 가능
- **참고**: 조직도는 부서명+인원수만 있어도 충분합니다 (실명 불필요)

## 아진산업 요청 체크리스트

### 최소 요청 (P0 — 5가지)

- [ ] 현대/기아에 실제 발송한 이메일 샘플 5건 (수신자명 마스킹)
- [ ] 8D 보고서 실제 양식 1건 (부품명/불량 내용 유지, 담당자명 마스킹)
- [ ] ECN 변경통보 실제 양식 1건
- [ ] 조직도 구조표 (본부 → 부서 → 팀, 각 팀 인원수)
- [ ] 사원번호 체계 규정 (부서별 접두어 규칙)

### 확장 요청 (P1 — 5가지 추가)

- [ ] 회사 소개서 또는 IR 자료 (PDF)
- [ ] 사내 약어집/용어 사전 (있다면)
- [ ] 주요 SOP 5건 (프레스/용접/검사 등)
- [ ] 신입사원 교육 자료 (OJT 자료)
- [ ] 보유 인증 목록 (IATF, ISO 등 + 유효기간)

> **위 10가지만 확보하면 현재 시뮬레이션 기반 시스템을 실데이터 기반 운영 시스템으로 전환할 수 있습니다.**
