## 2. 기능 A: 인원 검색 및 조직도

### SILLI 부합성: **업무 효율화**
649명 조직에서 "이 업무 담당자가 누구지?"는 가장 빈번한 질문. 검색 시간 단축 = 직접적 업무 효율화.

### 구현 상태 (95%)

| 항목 | 상태 | 비고 |
|------|------|------|
| FTS5 + ChromaDB 하이브리드 검색 | ✅ 완전 | RRF 통합 정렬 |
| ML 의도분류 (TF-IDF+LR) | ✅ 완전 | 5ms 응답, LLM 불필요 |
| 검색 이력 (20건) + 5종 정렬 | ✅ 완전 | 관련도/이름/부서/직급/사업장 |
| HTML 카드형 조직도 | ✅ 완전 | 부서별 + 전체 |
| Folium 사업장 지도 | ✅ 완전 | 19개소 표시 |

### 아진산업 적합성 평가
- **강점**: 12개 국내 사업장 분산 인력에서 시맨틱 검색은 실질적 가치. 부서 약어/별칭 자동 매칭.
- **현실 반영**: 사원 329명(시뮬레이션) — 실제 649명 대비 약 50%. 데모용으로 충분하나 실제 도입 시 전체 데이터 필요.

### 개선 제안
- **검색 분석**: 가장 많이 검색되는 부서/역할 통계 → 조직 운영 인사이트

### 불필요 요소: **없음** — 핵심 기초 기능으로 전 직원 필수

## 3. 기능 B: 문서 검색 및 작성

### SILLI 부합성: **업무 효율화 + 품질 개선**
8D, ECN, PPAP 등 OEM 제출 문서의 품질 = 납품 자격 유지. 자동 생성 + 품질 평가 = 반려율 감소.

### 구현 상태 (85%)

| 항목 | 상태 | 비고 |
|------|------|------|
| 문서 유형 15종 | ✅ 완전 | 사내 2 + 사외 13 (8D/ECN/PPAP 등) |
| Few-shot RAG | ⚠️ 코드 완전, 데이터 확인 필요 | ChromaDB 인덱싱 대상 문서 확인 필요 |
| 문서 품질 자동 평가 (5기준) | ✅ 완전 | 구조/분량/전문성/완성도/톤 |
| 버전 비교 diff | ✅ 완전 | 추가/삭제/변경 줄 수 + HTML 하이라이트 |
| CC 자동 추천 | ✅ 완전 | 문서유형별 필수/권장/선택 10종 |
| 내보내기 (DOCX/PDF/ODT) | ✅ 완전 | v3.3에서 HWPX→ODT 전환 |

### 아진산업 적합성 평가
- **강점**: 현대/기아 SQ 제출 문서(PPAP, 8D)를 아진 스타일로 자동 생성 — **실무 임팩트 최고**.
- **핵심 가치**: 신입이 양식을 찾고 작성법을 익히는 시간(하루 1~2시간) 절감.

### 개선 제안
- **Few-shot RAG 데이터 검증**: `data/documents/` 디렉토리의 실제 문서 수 확인 → 인덱싱 동작 보장
- **문서 미리보기**: Jinja2 렌더링 결과를 작성 전에 프리뷰로 표시

### 불필요 요소: **없음** — 제조업 핵심 업무

## 4. 기능 C: AI 업무 도우미

### SILLI 부합성: **생산성 향상** (핵심 취지)
신입 온보딩 3~6개월 → 수주 내 전력화. SILLI가 가장 중시하는 "생산성 향상"에 직접 대응.

### 구현 상태 (90%)

| 항목 | 상태 | 비고 |
|------|------|------|
| SOP 8종 | ✅ 완전 | 설비 3 + 업무 프로세스 5 |
| 협업 시나리오 5종 | ✅ 완전 | 8D/ECN/SPC/PPAP/안전점검 |
| 듀얼 모드 (교육/업무) | ✅ 완전 | 모드별 응답 스타일 분기 |
| LLM 스트리밍 | ✅ 완전 | Ollama + 네비 차단 |
| 퀴즈 자동 생성 + 재학습 | ✅ 완전 | related_step + "Step N 다시 보기" |
| 부서 selectbox 고정 | ✅ 완전 | SYS_ADMIN/HR_ADMIN만 변경 가능 |
| HUD 진행률 바 | ✅ 완전 | 색상 코딩 (빨/주/초) |
| 데모 빠른 질문 | ✅ 완전 | 생산기술팀 6개 (SOP+협업 트리거) |

### 아진산업 적합성 평가
- **강점**: "품질팀에서 8D 올려달라는데?" → LLM 없이 즉시 협업 가이드 응답. 신입이 **선배에게 묻지 않아도** 되는 가치.
- **SILLI 데모 핵심**: 프레스 트라이 SOP → 퀴즈 → 오답 재학습의 **완전한 학습 루프** 시연 가능.

### 개선 제안
- **퀴즈 결과 → 커리큘럼 반영**: 오답 분야를 커리큘럼 우선순위에 반영하는 적응형 학습
- **첫 주 커리큘럼 데이터**: `FIRST_WEEK_CURRICULUM` 구체적 일별 콘텐츠 추가 (현재 프레임만 존재)

### 불필요 요소: **없음** — 경진대회 핵심 취지와 완벽 일치

## 5. 기능 D: 법규/규정 모니터링

### SILLI 부합성: **안전성 향상 + 투자비 절감**
법규 미준수 = 납품 자격 박탈 + 과징금. 외부 컨설팅(1,000만원+) 대체 효과.

### 구현 상태 (85%)

| 항목 | 상태 | 비고 |
|------|------|------|
| 데모 시나리오 3종 | ✅ 완전 | 산안법(85점)/관세(78점)/REACH(52점) |
| 원 페이지 대시보드 | ✅ 완전 | 4메트릭 + TOP-3 카드 |
| 시뮬레이션 결과 카드 | ✅ 완전 | 변경 비교 + 리스크 + 시설 매핑 |
| 리스크 스코어링 | ✅ 완전 | 영향도 40 + 가능성 30 + 긴급도 30 = 100점 |
| 관세 시뮬레이터 | ✅ 완전 | 슬라이더 0~50% + 6품목 원가 영향 |
| 10종 크롤러 | ✅ 코드 완전 | ISO/MSDS/EU/국내법/OEM/APQP/ESG/EV/Trade/미국 |
| 오프라인 데모 | ✅ 완전 | 네트워크 없이 시나리오 내장 데이터로 동작 |

### 아진산업 적합성 평가
- **강점**: "산안법 안전거리 300mm→400mm" 시나리오가 **아진산업 프레스 라인에 직접 해당**. 데모 임팩트 극대.
- **핵심 가치**: 트럼프 관세 25% → JOON INC(조지아) 공급 원가 400억 원 영향 — **실제 경영 이슈**.

### 개선 제안
- **실시간 크롤러 데모**: 네트워크 환경에서 실제 크롤링→변경 감지 라이브 시연 (현재 오프라인 시나리오만)
- **영향 네트워크 시각화**: 시뮬레이션 결과에 Plotly 네트워크 그래프 연동 (기존 `impact_network.py` 활용)

### 불필요 요소: **없음** — 1차 협력사의 생존 이슈

## 6. 기능 E: 인사 관리

### SILLI 부합성: **투자비 절감** (보안 자동화)
AI 시스템의 접근 제어 + 보안 감사 자동화. 직접적 비용 절감보다 **시스템 안전성 확보** 역할.

### 구현 상태 (78%)

| 항목 | 상태 | 비고 |
|------|------|------|
| 7탭 Admin UI | ✅ 완전 | 사용자/생성/이력/보안/분석/통계/도구 |
| 본부/부서 cascade 수정 | ✅ 완전 | v3.4 레이스 컨디션 해소 |
| 보안 감사 (3종 감지) | ✅ 완전 | 무차별대입/야간접근/비활성계정 |
| AI 활용 분석 + ROI | ✅ 완전 | 기능별 사용량 + 비용 절감 추정 |
| 비밀번호 정책 강화 | ✅ 완전 | 8자+6조건+실시간 강도 표시 |
| 테스트 계정 | ⚠️ 부분 | 12계정 확인 (문서상 33명 기재) |

### 아진산업 적합성 평가
- **강점**: RBAC 6단계 + 28개 세부 권한 → 부서별 정보 접근 통제 체계적.
- **현실 반영**: 649명 조직의 권한 관리 자동화는 실질적 필요.

### 개선 제안
- **테스트 계정 33명 확보**: 현재 12명 → 문서 기재 33명에 맞춰 추가 시딩 필요
- **인력 통계 차트 강화**: 본부별/직급별/근속연수 분석은 데모에서 시각적 임팩트 높음

### 불필요 의심 요소
- **ROI 산출 기능**: AI 시스템의 ROI를 AI 시스템 자체가 계산하는 것은 **자기 참조적**. 데모에서 설득력이 약할 수 있음. 다만 "경영진 보고용 대시보드"로 포지셔닝하면 유효.

## 7. 기능 F: 설비/공정 AI

### SILLI 부합성: **품질 개선 + 안전성 향상** (제조 현장 AI의 핵심)
"관리 한계 안에 있어 사람 눈으로는 정상이지만, AI가 패턴을 잡는다" — SILLI 취지와 가장 직접적 부합.

### 구현 상태 (92%)

| 항목 | 상태 | 비고 |
|------|------|------|
| 4탭 구조 (개요/매뉴얼/SPC/검사) | ✅ 완전 | v3.4 SPC 탭 신설 |
| 개요 5하위탭 | ✅ 완전 | 설비개요/긴급조치/장비유형/예측정비/ML엔진 |
| 매뉴얼 AI 3하위탭 | ✅ 완전 | 에러코드/증상가이드/AI질의 |
| 에러코드 DB 201건 | ✅ 완전 | 7종 설비 유형 |
| ML 에러 검색 (TF-IDF) | ✅ 완전 | 동의어 79개 + 장비 가중치 |
| 에러 발생 이력 DB 685건 | ✅ 완전 | 심각도별 복구시간 차등 |
| Markov 연쇄 고장 예측 | ✅ 완전 | 70+ 인과규칙, DFS 3단계 |
| SPC Nelson 8 Rules | ✅ 완전 | 가이드 딕셔너리 + 차트 Annotation |
| 5공정 건강 대시보드 | ✅ 완전 | 신호등 + Cpk + 위반 수 |
| XGBoost 금형 수명 예측 | ✅ 완전 | 10특성, 배스터브 곡선 |
| MTBF 예측 정비 | ✅ 완전 | 15대 기계, 240건 수리이력 |
| 증상 카테고리 드롭박스 | ✅ 완전 | 7장비 40카테고리 |
| 검색 피드백 (👍/👎) | ✅ 완전 | search_feedback.db |
| 점검 체크리스트 | ✅ 완전 | 3종 장비 x 3주기 = 9템플릿 |

### 아진산업 적합성 평가
- **강점**: **실제 ML 모델 4종 동작** (TF-IDF, Isolation Forest, XGBoost, Markov Chain) — "PPT AI"가 아닌 실동작 시스템.
- **핵심 가치**: 에러코드를 모르는 신입도 "프레스에서 이상한 소리가 나요" → AI가 유사 에러코드 + 이력 + 연쇄 경고 제공.
- **SILLI 데모 최강**: SPC 탭에서 Nelson Rule 위반 음영 + 풍선 → "사람이 못 보는 패턴을 AI가 감지" 시연.

### 개선 제안
- **SPC 실시간 연동**: 현재 합성 데이터 → 실제 MES/센서 데이터 연동 시 실시간 모니터링 가능
- **매뉴얼 RAG 데이터**: `data/equipment/manuals/` 디렉토리에 실제 장비 매뉴얼 PDF 인덱싱 필요
- **모바일 점검**: 현장에서 태블릿으로 점검 체크리스트 입력 → 현재 데스크톱 UI 최적화

### 불필요 의심 요소
- **도면 검색 (제거됨)**: v3.3에서 이미 삭제됨. 코드 잔류(`drawing_search.py`) — 완전 삭제하거나 매뉴얼 AI에 통합 가능.
