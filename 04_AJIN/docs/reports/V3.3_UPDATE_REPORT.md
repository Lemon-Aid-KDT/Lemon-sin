## 0. v3.3 업데이트 요약

### 핵심 변경사항
- **보안 취약점 수정**: 세션 폴백 자동 로그인 제거 (admin 세션 탈취 방지)

## 1. 보안 수정

### 1-1. 자동 로그인 취약점 수정 (Critical)

| 항목 | 내용 |
|------|------|
| **파일** | `core/auth/session_store.py` |
| **문제** | `get_session_id_from_cookies()` 에서 쿠키 미발견 시 서버의 최근 세션 파일을 자동 로드 → admin 로그인 후 다른 사용자가 admin으로 자동 인증 |
| **원인** | 단일 사용자 개발 환경용 폴백 로직 (`_get_latest_session_id()`)이 다중 사용자 환경에서 보안 위협 |
| **수정** | 폴백 로직 제거 — 쿠키에 `ajin_sid` 없으면 빈 문자열 반환 → 로그인 페이지 이동 |
| **영향** | 정상 로그인 사용자: 쿠키 기반 세션 복원 (기존과 동일). 쿠키 없는 접속: 로그인 필수 |

### 1-2. 비밀번호 정책 안내 강화

| 항목 | 내용 |
|------|------|
| **파일** | `ui/page_login.py`, `ui/page_profile.py` |
| **Before** | "6자 이상" 안내, `validate_password_strength()` 미호출 |
| **After** | HUD 스타일 조건 박스 (8자+대소문자+숫자+특수문자+연속3회 금지) + 실시간 강도 표시기 + 서버 사이드 검증 |

**실시간 강도 표시 (`_render_password_strength_indicator`)**:
- 6개 조건 각각 ✓/✗ 실시간 표시
- 진행 바: 빨강(약함) → 주황(보통) → 초록(강함)
- 적용 위치: 최초 로그인 강제 변경 / 로그인 페이지 PW 변경 / 프로필 PW 변경 (3곳 통일)

## 6. 예측 정비 엔진 (Feature F)

### 6-1. 신규 모듈

| 항목 | 내용 |
|------|------|
| **파일** | `features/equipment/maintenance_predictor.py` (359줄, 신규) |
| **기능** | 수리 이력 기반 MTBF(평균 고장 간격) 분석 + 계절별/주기별 고장 패턴 도출 + 다음 정비 시점 예측 |
| **데이터** | 가상 2년치 수리 이력 (2024.04~2026.04) 240건, 15대 기계 |
| **DB** | `data/equipment/maintenance.db` (SQLite) |

### 6-2. 분석 기능

| 기능 | 설명 |
|------|------|
| **MTBF 계산** | 기계별 고장수리 타임스탬프 간격 평균 + 표준편차 |
| **계절 패턴** | 봄/여름/가을/겨울별 고장 비율 (%) — 프레스: 여름 33.3% 집중 |
| **다음 정비 예측** | 마지막 수리일 + MTBF = 예상 다음 정비일, D-day 표시 |
| **위험도 분류** | 긴급(D-day 초과), 주의(14일 이내), 정상 |
| **비용 분석** | 기계별 평균 수리 비용 + 누적 비용 TOP 5 |
| **계절 인사이트** | 현재 계절 주의 대상 기계 자동 안내 |

### 6-3. UI 통합

| 항목 | 내용 |
|------|------|
| **위치** | `page_equipment.py` 개요 탭 → "예측 정비" 섹션 (ML 경고 아래) |
| **메트릭 카드** | 모니터링 기계 / 주의 필요 / 7일 이내 / 30일 이내 |
| **계절 경고** | 현재 계절 고장 빈도 높은 설비 자동 표시 |
| **상세 테이블** | 기계별 MTBF, 마지막 수리, 다음 예상, 남은 일수, 위험도, 평균 비용 |
