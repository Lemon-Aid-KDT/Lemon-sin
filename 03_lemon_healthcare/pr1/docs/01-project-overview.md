## 2. 발주 배경

(주)레몬헬스케어는 2026년 코스닥 상장(2026.03.26 승인)을 앞두고 B2C 건강관리 플랫폼 **'건강의신'** 으로 사업 포트폴리오를 확장 중이다. 기존 '청구의신'(실손보험 간편청구 앱)은 2025년 12월 31일자로 종료되었고, 국가 단위 실손보험 청구 플랫폼 '실손24'가 청구 기능을 흡수했기 때문이다.

회사는 청구의신을 통해 축적한 **770만+ 사용자 베이스**를 건강관리 플랫폼으로 피벗시켰으며, 이번 과제는 건강의신 서비스의 핵심 AI 알고리즘을 정의·구현하는 작업이다.

> 🔍 **상세 배경·문제 정의**: [02-background-problem.md](./02-background-problem.md)

## 3. 과제 명세 (Input / Output)

### 🟢 Input — 7가지

| 카테고리 | 항목 | 데이터 형태 | 비고 |
|---------|------|-----------|------|
| 인구통계 | 성별, 나이 | 범주형, 정수 | 필수 |
| 신체 정보 | 키(cm), 몸무게(kg) → BMI | 실수 | 필수 |
| 활동 정보 | 일일 걸음수 | 정수 | HealthKit/Health Connect 자동 수집 |
| 식단 | 섭취 음식 목록 | 텍스트 또는 이미지 | **과제 #2** |
| 영양제 | 복용 중인 영양제 | 이미지 (Supplement Facts) | **과제 #1** |
| 권장 기준 | KDRIs (한국영양학회) | 참조 DB | 룩업 테이블화 |
| (확장) | 심박수, 만성질환 | 웨어러블·문진 | v2~v4 활동점수용 |

### 🔵 Output — 5가지

| 출력 | 설명 | 가이드 매핑 |
|------|------|-----------|
| ① 부족 영양소 추천 | 현재 식단·영양제 대비 결핍 영양소 식별 | Ch.02 (KDRIs 30종) — 알고리즘 미정의 |
| ② 영양소 섭취량 권고 | 개인별 일일 권장 섭취량 | Ch.02 |
| ③ 체중 변화 예측 | 1주 / 1개월 / 3개월 후 체중 | Ch.03 (7-step) |
| ④ 활동(운동) 권고 | 권장 걸음수 + 활동점수 | Ch.01 (v1~v4) |
| ⑤ 목적별 특화 분석 | 눈건강 / 간기능 / 피로회복 | 별도 정의 필요 |

## 4. 회사 정의 산출식 요약

레몬헬스케어가 가이드 PPTX에서 명확히 정의한 산출식은 3개 챕터로 구성된다.

### Ch.01 활동점수 (v1 ~ v4)

```
v1 (기본걸음점수)
  → v2 (+ 심박수 강도 가중)
    → v3 (+ 그룹 백분위 보너스)
      → v4 (+ 만성질환 가중)
```

- **v1**: 권장걸음수 = 8,000 × 성별계수 × 나이계수 × BMI계수
- **v2**: v1 × (0.7 + 0.3 × 심박계수)
- **v3**: v2 + 백분위 보너스 (상위 10/20/30% → +10/+5/+3점)
- **v4**: v3 × 만성질환가중 (최대 ×1.3)
- **최종 상한**: 100점

### Ch.02 영양 기준 (KDRIs 2020)

- BMI 한국·아시아 분류 (저체중/정상/과체중/비만1/비만2)
- 30종 영양소 (에너지·거대영양소·미네랄·비타민)
- BMI별 칼로리 조정: −700 ~ +500 kcal

### Ch.03 체중 예측 7-step

1. BMR (Mifflin-St Jeor)
2. TDEE = BMR × 활동계수 (1.2 ~ 1.9)
3. 일일 수지 = 섭취칼로리 − TDEE
4. N일 누적 = Σ(일일 수지)
5. 이론 변화 = 누적 ÷ 7,700 kcal/kg
6. 현실 보정 (감량 ×0.85 / 증량 ×0.95)
7. 예측 체중 = 시작 체중 + 보정 변화

> 🔍 **알고리즘 상세**: [07-core-algorithm.md](./07-core-algorithm.md)

## 5. 갭 분석 — 우리가 직접 정의해야 할 영역

| 항목 | 회사 가이드 | 우리가 추가 정의·구현 |
|------|------------|---------------------|
| 활동점수 (v1~v4) | ✅ 산출식 명확 | 그룹 비교용 코호트 DB 설계 |
| 권장 영양소 | ✅ 19~49세 기준 | 50대+, 청소년, 임신부 확장표 |
| 체중 예측 | ✅ 결정론적 공식 | 사용자 적응형 보정계수 학습 모델 (Phase 3) |
| **부족 영양소 추천** | ❌ 알고리즘 미정의 | 식단→영양소 매핑 + 결핍 진단 로직 |
| **목적별 분석 (눈/간/피로)** | ❌ 정의 없음 | 의학 가이드라인 기반 영양소 매트릭스 |
| **식단 입력 처리** | ❌ 미정의 | 텍스트/이미지 → 영양소 변환 파이프라인 |
| **영양제 분석** | ❌ 미정의 | OCR + LLM 파싱 + 식약처 DB 매칭 |

> 💡 **5개 출력 중 4개의 핵심 가치 영역(부족 영양소·목적별 분석·식단·영양제)이 우리가 직접 알고리즘을 설계해야 하는 영역**이다. 이것이 사실상 프로젝트의 진짜 도전 영역이다.

## 6. 프로젝트 범위 (Scope)

### ✅ 포함 (In Scope)

- 5종 출력의 백엔드 산출식 구현 (Python)
- 영양제 라벨 OCR + LLM 파싱 파이프라인
- 식단 텍스트 매칭 (Phase 2) 및 이미지 인식 (Phase 3)
- 모바일 앱 프로토타입 (Flutter, iOS + Android)
- HealthKit / Health Connect 걸음수 자동 수집
- 5종 출력 UI (사용자 친화적 시각화)
- 면책 고지 표준 문구 + 사용자 동의 UI

### ❌ 제외 (Out of Scope)

- 디지털치료기기(DTx, SaMD) 인증 — 식약처 별도 절차
- LDB 의료데이터 직접 연동 — 인터페이스 설계까지만
- ISMS-P 인증 — 본사 차원 작업
- 영양제 e-커머스 (구매·결제) 기능
- 의료진 처방·진단 기능

## 7. 핵심 도전과제

1. **데이터 통합**: 식약처 영양 DB + AI Hub 음식 이미지 + 웨어러블 + 사용자 입력
2. **식단 → 영양소 변환 정확도**: 한국 음식 인식, 양 추정, 식품명 정규화
3. **개인 맞춤화**: 나이/성별/BMI/만성질환별 동적 기준값 적용
4. **체중 예측 신뢰도**: 결정론 공식 + 적응형 보정 ML
5. **법적 리스크 회피**: 의료법·약사법·건강기능식품법 (진단/처방 표현 금지)
6. **데이터 보안**: 건강정보=민감정보, 개인정보보호법 + ISMS-P 컴플라이언스
7. **차별화 확보**: 필라이즈 등 선행 주자 대비 차별점 (의료기관 연계 가능성)
