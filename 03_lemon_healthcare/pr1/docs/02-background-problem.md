## 1. 사회적·시장적 맥락

### 1.1 왜 지금 헬스케어 AI인가

| 트렌드 | 사실 | 본 프로젝트와의 연관성 |
|--------|------|---------------------|
| **고령화·만성질환 증가** | 한국 65세 이상 인구 비율 2025년 20% 돌파 (초고령사회 진입) | v4 만성질환 가중 알고리즘의 의의 |
| **포스트 코로나 건강 관심** | 건강기능식품 시장 2024년 6조 원 돌파, 연 8% 성장 | 영양제 분석 수요 ↑ |
| **웨어러블 보급** | Apple Watch, 갤럭시 워치 등 헬스 데이터 수집 디바이스 대중화 | HealthKit/Health Connect 연동 가능 |
| **의료 마이데이터 정책** | '건강정보 고속도로' 사업 추진 | 향후 LDB 연계 가능성 |
| **생성형 AI 보편화** | LLM 기반 멀티모달 분석이 모바일 앱에 적용 가능한 수준 도달 | OCR + LLM 파이프라인 실현 가능 |

### 1.2 한국 영양제·헬스케어 앱 시장 동향

- 영양제·건강기능식품 e커머스 매출 매년 두 자릿수 성장
- 사용자가 평균 **3~5종**의 영양제를 동시 복용 (중복·과다 섭취 위험 ↑)
- 건강 검진 기록 → 영양제 → 식단 → 운동을 **하나의 앱에서 통합**하는 서비스 수요 급증
- 필라이즈 등 선행 주자가 이미 시장에 진입, **추격이 아닌 차별화** 필요

## 2. 발주처 배경 — 레몬헬스케어의 전략적 위치

### 2.1 회사 정체성

(주)레몬헬스케어는 **LDB(Lemon Digital Bridge)** 라는 실시간 양방향 의료데이터 중계 플랫폼을 핵심 기술로 보유한 기업이다. 단순한 앱 회사가 아니라 **"AI가 학습할 수 있는 고품질 의료 데이터를 유통하는 인프라 기업"** 이라고 자사 정체성을 규정하고 있다.

| 항목 | 수치 |
|------|------|
| 누적 다운로드 | **770만+** (레몬케어 기준) |
| 상급종합병원 점유율 | **80%** (37개 / 47개) |
| 제휴 의료기관 | 130여 곳 |
| 표준화 의료데이터 API | 180여 종 |
| 2024년 매출 | 약 149억 원 (전년 대비 2.4배 ↑) |

### 2.2 사업 전환의 배경 — 청구의신 → 건강의신

```
2020              2024.10              2025.12              2026.03
청구의신 런칭  →  실손24 출시      →   청구의신 종료   →   코스닥 상장 승인
  (B2C 청구앱)    (국가 차원 흡수)     (서비스 종료)        + 건강의신 출범
```

- **청구의신**: 실손보험 간편청구 앱. 누적 760만+ 청구 처리
- **실손24**: 2024.10 보험개발원 운영 국가 청구 플랫폼 출시 → 청구 기능 흡수
- **결과**: 청구의신은 종료되었으나 **770만 사용자 베이스는 자산으로 남음**
- **전략**: 이 사용자 풀을 **건강의신** 이라는 B2C 건강관리 플랫폼으로 피벗

### 2.3 IPO와 본 프로젝트의 관계

- 2025년 11월 코스닥 상장 예비심사 청구 (2차 도전)
- 2026년 03월 26일 상장 승인
- 공모 자금 사용 계획: **B2C 진출, AI R&D, 글로벌 확장**

→ **본 프로젝트는 그 핵심 AI R&D의 일부**다. 즉, 레몬헬스케어 입장에서는 단순한 학생 협업이 아니라 **상장 이후 신성장 동력의 한 축**을 검증하는 작업이다.

## 3. 핵심 문제 정의

본 프로젝트는 세 가지 관점에서 문제를 정의한다.

### 3.1 사용자 관점 — 끊어진 건강 데이터의 통합

| 페인 포인트 | 현실 | 결과 |
|------------|------|------|
| **영양제 정보 비대칭** | 제품마다 성분 표기 형식이 다르고 영문 라벨도 많음 | 무엇을 얼마나 먹는지 파악 어려움 |
| **중복·과다 섭취 위험** | 평균 3~5종 동시 복용, 같은 성분이 여러 영양제에 중복 | 의도치 않은 과량 섭취 |
| **영양 균형 파악 어려움** | 식단(음식) + 영양제(보충)를 통합 분석하는 도구 부재 | 결핍/과잉을 알 수 없음 |
| **인과관계 불투명** | 운동량 ↑ 했는데 체중이 안 빠짐, 왜? | 동기 저하 |
| **만성질환자 맞춤 부재** | 일반인 기준 권장량만 제시 | 정확도 ↓ |
| **데이터 분산** | 걸음수는 헬스앱, 식단은 다이어트앱, 영양제는 또 다른 앱… | 통합 인사이트 부재 |

### 3.2 회사 관점 — 770만 사용자 유지·전환 과제

- **유지 과제**: 청구의신 종료 후 770만 사용자 이탈 방지
- **전환 과제**: B2B(병원·보험사) 매출 의존 구조에서 B2C로 사업 다각화
- **차별화 과제**: 필라이즈 등 시장 선점자 대비 어떻게 다를 것인가
- **데이터 축적 과제**: 향후 DTx(디지털치료기기)로 진화할 수 있는 의료급 데이터 자산을 구축

### 3.3 산업 관점 — 차별화 압박

- **필라이즈** (75만 MAU, 120억 투자 유치): 영양제 조합 분석 카테고리 1위
- **마이약, 홈스픽, CalZen, 식사진** 등 세분화된 경쟁자
- 단순한 "영양제 스캔 + 부족 영양소 추천" 만으로는 진입 불가
- **레몬헬스케어의 무기**: 상급종합병원 80% 네트워크 + LDB 의료데이터 + 770만 검증된 사용자

## 4. 문제 해결 가설

본 프로젝트는 다음 가설들이 참이라는 전제 위에 설계된다.

### 가설 1 — 사진 한 장으로 영양제 분석의 진입장벽을 제거할 수 있다
- **OCR + LLM 조합**으로 라벨 텍스트를 정확하게 구조화 가능
- 사용자는 검색·입력 없이 사진 촬영만으로 등록 → **마찰(friction) 최소화**
- 검증 지표: OCR 정확도 ≥ 85%, 사용자 등록 시간 ≤ 30초

### 가설 2 — 회사 가이드(v1~v4, KDRIs, 7-step)는 임상적으로 타당하다
- v1~v4 활동점수: 만성질환자 보정까지 포함하는 정교한 구조
- KDRIs 2020: 보건복지부 공식 한국인 영양 기준
- Mifflin-St Jeor: 임상 표준 BMR 공식
- 검증 지표: 의료자문위 검토 통과

### 가설 3 — 모바일 환경에서 5종 출력을 한 화면에 통합 제시할 수 있다
- HealthKit/Health Connect 자동 연동으로 사용자 입력 부담 최소화
- 시각적 대시보드 + 권고 카드 형태로 직관성 확보
- 검증 지표: 베타 테스트 SUS(System Usability Scale) ≥ 70점

### 가설 4 — 의료기관 연계 가능성이 차별화 핵심 가치가 될 수 있다
- 필라이즈 등 경쟁자는 사용자 입력 데이터에만 의존
- 레몬헬스케어는 LDB를 통해 **실제 진료·처방 데이터** 연결 가능 (향후)
- 검증 지표: LDB 연동 인터페이스 설계서 완성

## 5. 본 프로젝트의 의의

### 5.1 학생 팀 관점 — 학습 가치

| 차원 | 학습 내용 |
|------|----------|
| **도메인 지식** | 영양학 (KDRIs), 운동생리학 (BMR/TDEE), 임상 가이드라인 |
| **AI 기술** | OCR, LLM 프롬프트 엔지니어링, 멀티모달 파이프라인 |
| **풀스택** | Python 백엔드 + Flutter 모바일 + 헬스 데이터 연동 |
| **협업** | GitHub 기반 팀 개발, PR 리뷰, 이슈 관리 |
| **컴플라이언스** | 의료법·약사법·건강기능식품법, 개인정보보호법 |
| **실무 경험** | 발주처와의 실제 협업, 산업계 발표·시연 |

### 5.2 회사 관점 — 협업 가치

- **외부 시각 검증**: 회사가 정의한 산출식이 외부 개발자에게 이해 가능하고 구현 가능한가
- **갭 영역 보완**: 회사 가이드가 정의하지 않은 4개 영역(부족 영양소·목적별 분석·식단·영양제) 알고리즘 후보 확보
- **인재 발굴 채널**: 우수 학생 팀과의 사전 접점

### 5.3 사회적 관점 — 데이터 헬스케어 생태계 기여

- **표현 가이드라인 수립**: 의료법 위반 없이 권고하는 표준 문구 사례 축적
- **데이터 거버넌스**: 건강 민감정보 처리의 모범 사례 제시
- **DTx 진화 가능성**: 향후 디지털치료기기로 발전할 수 있는 기반 마련

## 6. 답해야 할 핵심 질문 (Research Questions)

본 프로젝트가 종료 시점에 답해야 할 4가지 핵심 질문이다.

### RQ1. 영양제 라벨 OCR + LLM 파싱 정확도가 실용 수준에 도달하는가?
- 측정: 100장 테스트셋에서 성분명·용량·단위 추출 정확도
- 목표: ≥ 85%
- 검증 방법: Phase 2 종료 시 정확도 리포트 작성

### RQ2. 한국인 BMI/만성질환 분기까지 포괄하는 산출식 구현이 가능한가?
- 측정: v1~v4 활동점수 + 7-step 체중예측 단위 테스트 통과율
- 목표: 100% (모든 분기 케이스 통과)
- 검증 방법: 회사 가이드 PPTX의 계산 예시와 일치 검증

### RQ3. 모바일 환경에서 5종 출력을 사용자 친화적으로 제시할 수 있는가?
- 측정: 베타 테스트 30명 이상의 SUS 점수
- 목표: ≥ 70점 (양호)
- 검증 방법: Phase 4 종료 시 사용성 평가

### RQ4. 의료법·약사법 표현 제약 안에서 가치 있는 권고가 가능한가?
- 측정: 법무 검수 시 진단·처방 표현 위반 건수
- 목표: 0건
- 검증 방법: Phase 3~4에서 법무팀(또는 의료자문위) 검토
