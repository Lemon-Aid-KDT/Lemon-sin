## 2. 의료법 (Medical Service Act)

### 2.1 핵심 금지 사항

본 프로젝트가 절대 해서는 안 되는 행위:

| # | 금지 행위 | 근거 |
|---|---------|------|
| ❌ | "OO질환입니다" 같은 **진단** 제공 | 의료법 §27 (무면허 의료행위) |
| ❌ | "OO약을 드세요" 같은 **처방** 제공 | 의료법 §27 |
| ❌ | "OO병원에 가세요" 외 **특정 의료기관 알선** | 의료법 §27의2 |
| ❌ | 의사·약사로 **오인되는 표현** | 의료법 §17, §27 |
| ❌ | 치료 효과를 **단정적으로 보장** | 의료법 §56 (의료광고 금지) |

### 2.2 허용되는 표현 (비의료 건강관리서비스)

| ✅ | 허용 행위 |
|---|---------|
| ✅ | KDRIs 기준 권장 섭취량 안내 |
| ✅ | 일반 식이·운동 권고 (질병 진단 없이) |
| ✅ | 영양소 결핍 가능성 안내 (질병명 X) |
| ✅ | "전문의·약사와 상담을 권장합니다" 안내 |
| ✅ | 일반인 대상 영양·운동·체중 정보 제공 |

### 2.3 면책 고지 표준 문구

#### 🏷️ 메인 디스클레이머 (모든 권고 화면)

```
본 서비스에서 제공하는 정보는 일반적인 건강 관리를
위한 참고 자료이며, 의사·약사·영양사의 전문적 진단
이나 처방을 대체하지 않습니다.

증상이 있거나 만성질환을 앓고 계신 경우, 반드시
전문가와 상담하시기 바랍니다.
```

#### 🏷️ 부속 디스클레이머 (영양제 화면)

```
영양제는 의약품이 아니며, 질병의 예방이나 치료를
보장하지 않습니다. 약을 복용 중이신 경우, 영양제와
의 상호작용에 대해 의료진과 상담하세요.
```

#### 🏷️ 부속 디스클레이머 (체중 예측 화면)

```
체중 변화 예측은 평균적인 수치를 바탕으로 산출되며,
개인의 대사·체질·생활습관에 따라 결과가 달라질 수
있습니다. 급격한 체중 변화는 건강에 해로울 수 있으
니 의료진과 상담하세요.
```

### 2.4 본 프로젝트 적용 체크리스트

- [ ] 모든 권고 화면 상·하단에 메인 디스클레이머 표시
- [ ] 영양제·체중 화면에 부속 디스클레이머 추가
- [ ] "진단", "처방", "치료" 단어 UI에서 사용 금지 (코드 검수)
- [ ] AI 응답에 진단 표현 검출 시 자동 차단 (LLM 시스템 프롬프트)
- [ ] 의료자문위 검토 시 표현 일관성 확인
- [ ] 법무 검수 시 진단 표현 0건 통과

## 3. 약사법 (Pharmaceutical Affairs Act)

### 3.1 핵심 금지 사항

| # | 금지 행위 | 근거 |
|---|---------|------|
| ❌ | **특정 의약품 추천**·이름 명시 | 약사법 §61, §68 |
| ❌ | **처방약 직접 알선** (대리 구매·배송) | 약사법 §47 |
| ❌ | 의약품을 **건강기능식품**처럼 광고 | 약사법 §63 |
| ❌ | 의약품의 효능·효과를 **사용자에게 직접 설명** | 약사법 §68 |

### 3.2 허용되는 표현

| ✅ | 허용 행위 |
|---|---------|
| ✅ | **건강기능식품 성분** 안내 (식약처 기능성 인정 원료) |
| ✅ | 영양제 라벨에 명시된 정보의 단순 표시 |
| ✅ | "약을 복용 중이라면 약사와 상의하세요" 안내 |
| ✅ | 일반의약품과 영양제의 차이점 일반 정보 |

### 3.3 본 프로젝트 적용

- 영양제 라벨 인식 결과는 **사용자가 가진 제품의 정보 표시**일 뿐 추천 X
- 결핍 영양소 권고 시 **성분명만 안내** (예: "비타민C 추가 섭취 권장") → 특정 제품·브랜드 X
- 처방약과의 상호작용 정보는 **공식 출처(식약처 의약품안전나라)에서만 인용**

### 3.4 체크리스트

- [ ] 특정 영양제·의약품 브랜드 추천 기능 없음 (검색·선택만)
- [ ] AI 응답에서 "OO 의약품"을 추천하는 텍스트 자동 차단
- [ ] 약사법 위반 가능 표현 검수 (Phase 3 법무 검수 시)
- [ ] 영양제 성분 정보는 식약처 건강기능식품 원료 DB만 사용

## 4. 건강기능식품법

### 4.1 핵심 사항

본 프로젝트는 영양제(=건강기능식품)를 다루므로 **건강기능식품에 관한 법률** 적용.

| # | 준수 사항 | 근거 |
|---|---------|------|
| ✅ | 식약처 **기능성 인정 원료**만 추천 (예: 루테인, 밀크씨슬, 코엔자임Q10) | 건기식법 §16 |
| ❌ | 식약처 미인정 원료의 기능성 표시 금지 | 건기식법 §18 |
| ❌ | "병이 낫는다", "즉효" 같은 과대광고 금지 | 건기식법 §18 |

### 4.2 본 프로젝트 적용

- 알고리즘 ⓓ (목적별 분석)에서 추천하는 모든 원료는 **식약처 기능성 인정 목록 매트릭스**에서만 선택
- 식약처 고시 기능성 표현 그대로 사용 (예: "간 건강에 도움을 줄 수 있음")

### 4.3 체크리스트

- [ ] 추천 원료 목록 = 식약처 건강기능식품 원료 DB와 일치
- [ ] 기능성 표현 = 식약처 고시 그대로 인용 (창작 금지)
- [ ] 원료별 일일 섭취량 = 식약처 기준 준수
- [ ] 흡연자 등 분기 처리 (예: 루테인 + 흡연자 경고)

## 5. 개인정보보호법

### 5.1 민감정보 분류

본 프로젝트가 수집하는 데이터의 법적 분류.

| 데이터 | 분류 | 별도 동의 필요 |
|--------|------|--------------|
| 이름, 이메일 | 일반 개인정보 | 일반 동의 |
| 나이, 성별, 키, 몸무게 | 일반 개인정보 | 일반 동의 |
| **만성질환 정보** | **민감정보 (건강 정보)** | ⭐ **별도 동의 필수** |
| **복약 정보** | **민감정보** | ⭐ **별도 동의 필수** |
| **검진 기록** (LDB 연계 시) | **민감정보** | ⭐ **별도 동의 필수** |
| 걸음수, 심박수 | 민감정보 (생체정보) | ⭐ 별도 동의 필수 |
| 영양제·식단 사진 | 일반 개인정보 (이미지) | 일반 동의 |

> ⚠️ **중요**: 민감정보는 일반 동의에 포함 불가. **별도 항목·별도 화면**으로 동의받아야 함.

### 5.2 별도 동의 UI 설계

#### 동의 화면 구조 (예시)

```
┌─────────────────────────────────────────────┐
│ ✅ [필수] 서비스 이용약관                     │
│ ✅ [필수] 개인정보 수집·이용 동의            │
│ ☐ [선택] 마케팅 정보 수신                   │
│                                              │
│ ─────────────────────────────────────       │
│  📌 민감정보 별도 동의 (각각 동의)            │
│ ─────────────────────────────────────       │
│                                              │
│ ☐ [필수] 건강 정보 (만성질환·복약) 수집·이용 │
│         💡 부족 영양소 분석에 사용됨          │
│ ☐ [필수] 생체 정보 (걸음수·심박수) 수집·이용 │
│         💡 활동점수·체중 예측에 사용됨        │

│                                              │
│         [취소]      [동의하고 시작하기]       │
└─────────────────────────────────────────────┘
```

### 5.3 가명정보 처리

만성질환 데이터를 통계·분석 목적으로 활용 시:

- **직접 식별자 제거**: 이름, 전화번호, 이메일, IP 주소
- **간접 식별자 일반화**: 생년월일 → 연령대 (10세 단위), 주소 → 시/도
- **결합 위험 평가**: 사용자 ID 해시, 외부 데이터와의 결합 가능성 검토

### 5.4 데이터 주체의 권리 (필수 구현 기능)

| 권리 | 구현 |
|------|------|
| **열람권** | 마이페이지 > 내 데이터 보기 |
| **정정·삭제권** | 마이페이지 > 데이터 수정/삭제 |
| **처리정지권** | 마이페이지 > 분석 일시 정지 |
| **동의 철회권** | 설정 > 동의 항목별 철회 |
| **회원탈퇴 시 데이터 즉시 삭제** | 백업도 90일 내 폐기 |

### 5.5 체크리스트

- [ ] 별도 동의 UI 구현 (3가지 민감정보 항목별)
- [ ] 회원가입 후 진입 시점에 동의 받기 (데이터 수집 전)
- [ ] 마이페이지에 데이터 열람·수정·삭제 기능
- [ ] 회원탈퇴 시 즉시 삭제 + 백업 폐기 절차

## 6. ISMS-P 인증

### 6.1 인증 개요

| 항목 | 내용 |
|------|------|
| **공식 명칭** | 정보보호 및 개인정보보호 관리체계 인증 |
| **운영 기관** | 한국인터넷진흥원 (KISA) |
| **인증 기준** | 101개 (관리체계 16 + 보호대책 64 + 개인정보 단계별 21) |
| **유효기간** | 3년, 매년 사후심사 |
| **소요 기간** | 갭 분석 → 컨설팅 → 심사 → 인증 (총 6~12개월) |

### 6.2 본 프로젝트의 ISMS-P 위치

| 항목 | 학생 단계 | 정식 출시 시 |
|------|---------|------------|
| 의무 인증 대상 | ❌ 아님 | △ (매출·이용자 기준 도달 시) |
| 권고 사항 | ✅ 자발적 | ✅ 강력 권고 (의료 데이터 신뢰도) |
| 본 프로젝트 범위 | 갭 분석 + 일부 통제 적용 | 본격 인증 추진 |

### 6.3 본 프로젝트가 적용할 핵심 통제

ISMS-P 101개 기준 중, 본 프로젝트가 우선 적용할 항목:

#### 관리체계 (16개 중 일부)

## 7. DTx 해당 여부 판단

### 7.1 DTx (디지털치료기기) 정의

식약처 기준:

```
다음을 모두 만족 시 → 디지털치료기기 (의료기기로 분류, 식약처 허가 필요)

┌─ ① 치료 작용기전의 과학적·임상적 근거가 있고
├─ ② 질병 예방·관리·치료를 목적으로 하며
└─ ③ 특정 질환자에게 증상 개선 프로토콜 제공
```

### 7.2 본 프로젝트의 DTx 해당성 판단

| 기준 | 본 프로젝트 | 결과 |
|------|-----------|------|
| ① 치료 작용기전 임상 근거 | ❌ (참고 정보만 제공) | DTx 아님 |
| ② 질병 예방·관리·치료 목적 | ❌ (일반 건강관리) | DTx 아님 |
| ③ 특정 질환자 증상 개선 프로토콜 | ❌ (일반 권고만) | DTx 아님 |

→ **결론: 본 프로젝트는 비의료 건강관리서비스**

### 7.3 비의료 건강관리서비스 포지셔닝

```
┌─ ✅ 영양·운동·체중 일반 권고만 제공
├─ ✅ 질병 치료 목적이 아니고
└─ ✅ 일반인 대상 웰니스 서비스
   → 비의료 건강관리서비스 → 식약처 허가 불필요
```

### 7.4 DTx 경계를 넘지 않기 위한 가드

향후 기능 추가 시 DTx로 오인되지 않도록:

- ❌ "당뇨 치료 프로그램" 같은 이름 X
- ❌ "고혈압 환자를 위한 처방 가이드" X
- ❌ "이대로 따라하면 OO병이 낫습니다" X
- ✅ "당뇨 위험군을 위한 영양 관리 가이드" → 가능
- ✅ "고혈압 관리 시 주의 영양소" → 가능 (정보 제공)

### 7.5 체크리스트

- [ ] 모든 기능이 "치료"가 아닌 "관리·참고"로 표현됨
- [ ] 특정 질환자용 증상 개선 프로토콜 부재
- [ ] 사전 법률 검토 (Phase 2~3)
- [ ] 보건복지부 비의료 건강관리서비스 가이드라인 준수
- [ ] 향후 DTx 진입 검토 시 별도 인허가 절차 (Year 3+)

#### 코드 레벨 표현 검출 (간단 정규식)

```python
FORBIDDEN_TERMS = [
    "진단", "처방", "치료", "확실히", "분명히",
    "보장", "이 약", "이 의약품", "처방전",
]

def check_forbidden_terms(text: str) -> list[str]:
    """금지 표현 검출"""
    found = [term for term in FORBIDDEN_TERMS if term in text]
    return found

# 사용
issues = check_forbidden_terms(ai_response.content)
if issues:
    logger.warning(f"Forbidden terms detected: {issues}")
    # 응답 재생성 또는 차단
```

다음 상황 발생 시 즉시 의료자문위에 알림:

| 트리거 | 우선순위 |
|--------|--------|
| 알고리즘 출력에 진단·처방 표현 발견 | 🔴 즉시 |
| 사용자 신고 (잘못된 권고) | 🔴 24시간 내 |
| 식약처·보건복지부 가이드라인 변경 | 🟡 1주일 내 |
| KDRIs 갱신 발표 | 🟡 즉시 (수치 갱신) |
| 새로운 만성질환 분류 추가 | 🟢 분기 내 |

### 11.4 체크리스트

- [ ] 위급 신호 감지 룰 정의 (LLM 프롬프트 + 룰 기반)
- [ ] 응급 자원 정보 페이지 구현 (1577-0199, 109, 1339)
- [ ] 데이터 유출 대응 절차서 작성
- [ ] 비상 연락망 (팀 + 회사 + KISA + 개인정보보호위원회)
